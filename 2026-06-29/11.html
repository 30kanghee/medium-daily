Stop Paying $20/month. Use NVIDIA Build: 80+ Free AI Models

Lorenzo Zarantonello · GoPenAI · 5분 읽기 · 원문 · freedium

요약

NVIDIA Build에서 API 키 하나로 80개 이상의 AI 모델을 무료로 쓸 수 있으며, VS Code의 Cline에서 완전한 에이전트 코딩이 가능하다.

NVIDIA NIM(Inference Microservices)은 build.nvidia.com을 통해 MiniMax, Kimi, DeepSeek 등 100개 이상의 AI 모델을 무료로 제공한다. API 키 하나만 발급받으면 OpenAI SDK와 동일한 방식으로 즉시 사용할 수 있다. Cursor는 무료 플랜에서 커스텀 모델을 선택할 수 없고, Pro 플랜에서도 에이전트 기능은 자체 백엔드에 묶여 있어 한계가 크다. 반면 VS Code의 오픈소스 확장인 Cline을 사용하면 파일 읽기·쓰기·터미널 명령 실행 등 완전한 에이전트 루프를 NVIDIA 무료 추론 레이어 위에서 돌릴 수 있다. 다만 모델별로 속도와 품질 차이가 크므로, 실제 사용 전에 function calling 지원 여부와 응답 속도를 반드시 확인해야 한다.

이 서비스의 이름은 NVIDIA NIM, 즉 NVIDIA Inference Microservices다. build.nvidia.com에서 접근할 수 있다.

카탈로그에는 100개 이상의 AI 모델이 있으며, 그중 상당수는 무료이고 NVIDIA 자체 DGX Cloud 인프라에서 호스팅된다. 이미 이름을 들어봤을 모델들이 포함돼 있다: MiniMax, Kimi, DeepSeek, GLM, GPT-OSS 등.

무료 API 키 발급받기

사실 이 부분이 가장 쉬운 단계다. build.nvidia.com에 접속해 무료 계정을 만들면 된다. 이메일과 전화번호 인증이 필요하다. 로그인 후 계정 설정에서 API 키를 생성한다. 한 번만 보여주므로 반드시 저장해 둘 것. 키는 다음과 같은 형태다:

nvapi-xxxxxxxxxxxxxxxx

가입 시 추론 크레딧 1,000개가 즉시 지급되며, 요청 시 최대 5,000개까지 받을 수 있다. 분당 최대 40건의 요청이 가능하고, 신용카드는 필요 없다. 진지한 프로토타입 제작에는 충분한 양이다.

모든 모델이 동일한 OpenAI 호환 엔드포인트를 노출한다는 점이 가장 큰 장점이다. 설정 파일에서 문자열 하나만 바꾸면, OpenAI API 형식을 지원하는 모든 도구에서 NVIDIA 모델을 바로 사용할 수 있다. 예를 들어 Kimi 모델의 페이로드는 다음과 같다:

// moonshotai/kimi-k2.6

const payload = {
  "model": "moonshotai/kimi-k2.6",
  "messages": [{"role":"user","content":""}],
  "max_tokens": 16384,
  "temperature": 1.00,
  "top_p": 1.00,
  "stream": stream,
};

익숙한 구조다. 즉, 코딩 도구, 에이전트, 스크립트 모두 설정 한 줄만 바꿔서 NVIDIA의 무료 추론 레이어로 연결할 수 있다는 뜻이다.

IDE가 무료 모델의 발목을 잡는다

일부 최신 모델은 완전히 무료이며 직접 다운로드도 가능하다. 그것은 문제가 아니다. 문제는 도구 측에 있다.

어떤 도구, IDE, CLI를 쓰느냐에 따라 결과가 크게 달라진다. 파일 읽기, 코드 작성, 터미널 명령 실행, 반복 수정까지 이어지는 완전한 에이전트 루프를 제공하는 도구가 있는 반면, 그냥 채팅 창만 주는 도구도 있다. 이 차이는 상당히 중요하다.

Cursor: 한 도구, 두 가지 문제

Cursor가 가장 먼저 떠오르는 선택지지만, 제대로 작동하지 않는다.

Cursor에서 설정 → Models로 이동해 OpenAI API 키 토글을 활성화하고, Override Base URL로 https://integrate.api.nvidia.com/v1을 입력한 뒤 nvapi- 키를 추가하면 NVIDIA 모델을 Cursor 안에서 쓸 수 있다.

문제 1: 무료 플랜은 커스텀 모델 선택 자체가 막혀 있다. Cursor 무료(Hobby) 플랜에서 커스텀 NVIDIA 모델을 선택하려 하면 다음 메시지가 뜬다: “Free plans can only use Auto. Switch to Auto or upgrade plans to continue.”

문제 2: Pro 플랜에서도 채팅 패널만 사용할 수 있다. Cursor는 Composer, 인라인 수정, 자동완성, 코드 실행 같은 에이전트 기능을 자체 백엔드에 묶어놓았다. Pro 등급에서도 이러한 기능들은 커스텀 엔드포인트를 통해 라우팅되지 않는다.

Cline — VS Code에서 완전한 에이전트 코딩

여기서부터 흥미로워진다. Cursor로 넘어간 사람들이 많지만, 무료 AI 모델을 쓰고 싶다면 Cline이 VS Code와 Cursor의 격차를 없애줄 수 있다.

Cline은 VS Code 사이드바에서 실행되는 오픈소스 AI 코딩 에이전트다. 설치 수 500만 이상, GitHub 스타 61,000개 이상을 보유하고 있으며, 원래 “Claude Dev”라는 이름으로 출시됐다가 지금은 완전히 모델에 종속되지 않는(model-agnostic) 구조로 바뀌었다. 참고로 Cline은 기본 무료 옵션으로 deepseek-v4-flash를 제공한다.

Cline에서 무료 AI 모델 사용하기

설정은 2분이면 충분하다. VS Code 또는 Cursor 마켓플레이스에서 Cline을 설치한다. Cline 설정을 열고 공급자(provider)를 “OpenAI Compatible”로 선택한다. Base URL을 https://integrate.api.nvidia.com/v1로 설정한다. nvapi- 키를 붙여 넣는다. NVIDIA 카탈로그에서 모델 ID를 입력한다(예: moonshotai/kimi-k2.6). 해당 모델의 API 키를 별도로 발급받는 것도 잊지 말아야 한다.

Cline은 전체 코드베이스를 읽고, 파일을 생성·수정하며, 터미널 명령을 실행하고, stderr를 읽어 오류를 디버깅하고, 작업이 완료될 때까지 반복한다. 이 모든 루프가 NVIDIA의 무료 추론 레이어 위에서 돌아간다.

한 가지 중요한 점: Cline의 에이전트 기능은 function calling에 의존한다. NVIDIA 카탈로그의 모든 모델이 tool use를 지원하는 것은 아니다. 에이전트 작업에 모델을 쓰기 전에 반드시 해당 모델 카드를 확인해야 한다.

저자가 직접 테스트한 결과, glm-5.1과 gemma-4-31b-it는 너무 느려서 실용적이지 않았다. moonshotai/kimi-k2.6은 처음엔 좋았지만 갑자기 중국어와 알 수 없는 텍스트를 뱉어내기 시작했다. 반면 openai/gpt-oss-20b는 꽤 쓸 만했다. 결론은 어느 정도 시행착오가 필요하다는 것이다.

월 20달러를 내지 않아도 진지한 프로토타입 제작이 가능하다. 다만 최고 성능 유료 모델과 동등하다고 말하기는 어렵다. 학습과 탐색, 사이드 프로젝트를 위한 현실적인 대안으로 보는 것이 맞다.
