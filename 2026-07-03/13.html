The NSA Just Published Its First MCP Threat Model. It Misses the Real Problem.

Tanmay Deshpande · Data Science Collective · 6분 읽기 · 원문 · freedium

NSA가 MCP의 보안 허점을 낱낱이 짚어낸 첫 위협 모델을 내놨지만, 정작 진짜 위험은 프로토콜 안이 아니라 그 이전 설계 단계에 있다는 지적이다.

요약

기존 API 보안은 클라이언트가 요청하고 서버가 응답하는 방향을 전제로 설계돼 왔지만, MCP는 이 구조를 뒤집어 서버가 클라이언트를 대신해 데이터를 조회하고 작업을 실행하게 만든다. 그래서 익숙하게 써오던 인증, 감사 로그, 최소 권한 같은 보안 사고방식이 MCP 환경에서는 방향 자체가 맞지 않는다는 것이 저자의 문제의식이다. NSA가 최근 내놓은 MCP 위협 모델은 이런 구조적 허점을 상당히 솔직하게 짚었는데, 프로토콜 차원에서 접근 제어가 필수가 아니고 도구 연결 시점에 역할 구분이 없으며 권한 구분이 강제되지 않는다는 점 등을 지적한다. 저자는 이 분석이 정확하지만 절반짜리 진단이라며, 사후 보완책으로는 근본 문제를 해결할 수 없고 진짜 해법은 설계 철학 자체를 바꾸는 데 있다고 주장한다.
