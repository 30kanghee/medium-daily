Intervewer: Add a Field to an Order Table with Over 1 Billion Records.

Umesh Kumar Yadav · Stackademic · 5분 읽기 · 원문 · freedium

예전엔 몇 시간 걸리던 대용량 테이블 컬럼 추가가 이제는 순식간에, 혹은 무중단으로 끝난다는 실무 회고.

요약

저자는 예전 회사에서 10억 행짜리 주문 테이블에 컬럼을 추가해야 했던 경험을 회고하며 글을 시작한다. 당시 MySQL 5.6에서는 방법이 없어 새벽 시간대에 테이블을 통째로 잠그고 4~6시간짜리 DDL을 돌리는 수밖에 없었다고 말한다. 지금이라면 두 가지 대안이 있는데, 하나는 MySQL 8.0.12부터 지원하는 Instant ADD COLUMN으로 실제 데이터를 다시 쓰지 않고 메타데이터에 기본값만 기록해두는 방식이라 조건만 맞으면 순식간에 끝난다. 다른 하나는 gh-ost로, 고스트 테이블을 만들어 데이터를 조금씩 복사하고 binlog로 실시간 동기화한 뒤 마지막에 테이블명만 바꿔치기해서 서비스 중단 없이 스키마를 변경하는 방식이다. 결론은 예전의 최후 수단이 지금은 표준 관행이 됐다는 것이다.
