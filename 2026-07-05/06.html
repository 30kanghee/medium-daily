Building a RAG Pipeline for 10M+ Documents With Near-Zero Hallucination

Fareed Khan · Level Up Coding · 37분 읽기 · 원문 · freedium

환각을 없애는 비결은 더 똑똑한 모델이 아니라, 근거 없으면 침묵하게 만드는 파이프라인 설계다.

요약

이 글은 방대한 문서에서 답을 찾는 RAG 시스템이 왜 자신만만하게 틀린 답을 내놓는지에서 출발해, 이를 막는 4단계 안전장치를 실제로 구현해본다. 먼저 dense 임베딩과 BM25 키워드 검색을 결합하고 리랭커로 걸러 정확한 근거 문단을 찾고, 답변은 반드시 인용된 문장 안에서만 생성하도록 강제한다. 그다음이 핵심인데, 생성된 답을 문장 단위로 쪼개 각각이 인용 근거와 실제로 일치하는지 별도의 검증 모델이 채점하고, 가장 약한 문장 하나라도 근거가 부족하면 답변 전체를 모르겠다는 응답으로 전환시켜버린다. 실험 결과 답하기 어려운 함정 질문 100개 중 98개를 정직하게 회피했고, 대신 답변 가능한 질문의 46%만 답하는 트레이드오프를 감수했다고 밝힌다. 벡터 인덱스는 1000만 개 규모에서도 검색 속도가 유지되지만, 실제 병목은 검색이 아니라 LLM 호출 자체이며 검증 모델의 정확도가 전체 시스템 신뢰도의 진짜 한계라고 솔직하게 인정하며 글을 마무리한다.
