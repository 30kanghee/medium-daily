LLM vs RAG vs MCP: I Finally Know When to Use Each One

Pravash · Level Up Coding · 8분 읽기 · 원문

LLM과 RAG, MCP를 언제 어떻게 구분해 써야 하는지 실전 관점에서 정리한 글.

요약

저자는 LLM, RAG, MCP라는 세 아키텍처를 각각 언제 써야 하는지 1인칭 경험을 바탕으로 정리한다. LLM은 학습 시점에 고정된 내부 지식을 활용하는 반면, RAG는 기존 문서나 지식베이스에 답변의 근거를 두는 데 초점을 맞춘다. MCP는 도구·API·워크플로를 통해 모델이 실시간 데이터를 가져오거나 실제 행동을 하도록 만드는 역할을 한다. 결국 중요한 질문은 “어떤 AI가 더 똑똑한가”가 아니라 “이 문제를 풀기 위해 AI가 실제로 무엇을 해야 하는가”라는 것이 글의 결론이다.
