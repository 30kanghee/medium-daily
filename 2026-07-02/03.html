Stop Using != null — This Is How Senior Developers Actually Think

Real World Java · 4분 읽기 · 원문 · freedium

null 체크를 많이 하는 게 아니라, 애초에 null이 나올 수 없게 설계하는 것이 시니어 개발자와 주니어의 진짜 차이라는 글.

요약

글은 시니어 개발자가 버그를 적게 만드는 이유가 조건문을 더 꼼꼼히 챙겨서가 아니라,애초에 계속 체크가 필요한 구조 자체를 만들지 않기 때문이라고 짚는다. 코드 곳곳에 != null을 흩뿌리는 대신, 경계 지점에서 값을 검증하고 Optional 같은 도구를 활용해 null이 시스템 안으로 들어올 여지 자체를 차단하는 설계를 강조한다. 결국 핵심은 방어적인 체크의 양이 아니라, null이 문제조차 되지 않도록 미리 설계하는 사고방식의 차이라는 것이다.
