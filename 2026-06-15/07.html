Claude Code Is a Mess, Until You Install This Official Plugin

Mouez Yazidi · Towards AI · 9 min read · 원문 · freedium

Claude Code를 설치하고도 막막했다면, 공식 플러그인 하나로 완전히 달라진다 — claude-code-setup이 코드베이스를 분석해 맞춤형 자동화를 추천해준다.

요약

대부분의 개발자들이 Claude Code를 설치하지만 프로젝트의 큰 그림을 파악하지 못한 채 실망하고 떠난다. Anthropic이 공식 제공하는 claude-code-setup 플러그인은 코드베이스를 읽기 전용으로 분석해 MCP 서버, 스킬, 훅, 서브에이전트, 슬래시 커맨드 등 5가지 범주에 걸쳐 가장 가치 있는 자동화를 추천해준다. 이 플러그인은 package.json과 언어 파일, 디렉토리 구조를 살펴 프로젝트 유형을 감지하고 기술 스택에 맞는 맞춤 제안을 생성한다. 설치는 Claude Code 내에서 단 한 줄 명령으로 완료되며, Claude Code 플러그인 생태계 전체(공식 마켓플레이스와 커뮤니티 마켓플레이스)에 대한 입구 역할을 한다. 플러그인을 신뢰하기 전에 반드시 출처를 확인하는 보안 습관도 필요하다.

Claude Code, 처음엔 혼란스럽다

Claude Code를 처음 설치한 대부분의 개발자들이 비슷한 경험을 한다. 터미널을 열고, claude를 실행하고, 몇 가지 명령을 내려본다. 그런데 뭔가 이상하다. 기대했던 것만큼 스마트하지 않다. 프로젝트의 맥락을 모르는 것 같다. 실망하고 그냥 예전 방식으로 돌아간다.

문제는 Claude Code 자체가 아니다. 문제는 설정이다.

Claude Code는 올바르게 설정되기 전까지는 그냥 또 하나의 AI 채팅 인터페이스처럼 느껴진다. 하지만 제대로 설정하면 — 프로젝트 구조를 이해하고, 올바른 도구가 연결되고, 워크플로우에 맞게 자동화가 구성되면 — 완전히 다른 경험이 된다.

여기서 Anthropic의 공식 플러그인인 claude-code-setup이 등장한다.

왜 Claude Code의 기본 상태는 "엉망"인가

Claude Code를 처음 실행할 때 모델은 프로젝트에 대해 아무것도 모른다. 어떤 언어를 쓰는지, 어떤 테스트 프레임워크를 사용하는지, 배포 파이프라인이 어떻게 생겼는지 — 아무것도 모른다.

물론 시간을 들여 CLAUDE.md 파일을 직접 작성하고, MCP 서버를 수동으로 구성하고, 커스텀 훅을 설정할 수 있다. 하지만 대부분의 개발자들은 그렇게 하지 않는다. 설정 방법을 모르거나, 어디서 시작해야 할지 모르거나, 그냥 귀찮기 때문이다.

결과적으로 Claude Code는 강력한 도구임에도 불구하고 잠재력의 10%만 발휘된 채 사용된다.

claude-code-setup 플러그인이란

claude-code-setup은 Anthropic이 직접 개발하고 유지 관리하는 공식 플러그인이다. 현재 89,000회 이상 설치되었으며, Anthropic 검증 마크를 받은 플러그인이다.

이 플러그인이 하는 일은 단순하지만 강력하다: 코드베이스를 분석하고, 맞춤형 Claude Code 자동화를 추천한다.

중요한 점은 읽기 전용(read-only) 모드로 동작한다는 것이다. 파일을 수정하거나, 코드를 작성하거나, 어떤 변경도 하지 않는다. 오직 분석하고 추천할 뿐이다. 실제 설정을 적용할지 말지는 개발자 본인이 결정한다.

무엇을 분석하나

플러그인은 다음을 살펴본다:

  • package.json — 사용 중인 라이브러리와 프레임워크
  • 언어 파일 확장자 — Python인지, TypeScript인지, Go인지 등
  • 디렉토리 구조 — 모노레포인지, 마이크로서비스인지, 단일 앱인지
  • 설정 파일들 — Docker, CI/CD, 테스트 설정 등

5가지 범주의 추천

분석 후 플러그인은 다음 5가지 범주에서 가장 가치 있는 추천을 제시한다:

  1. MCP 서버 — 외부 도구 연동 (예: React 프로젝트에는 Playwright MCP 추천)
  2. 스킬 — 패키징된 전문 능력 (예: 코드 리뷰, 보안 검토)
  3. 훅 — 자동 트리거 액션 (예: 커밋 전 린팅, 테스트 실행)
  4. 서브에이전트 — 특화된 검토자 (예: 인증 코드 감지 시 보안 리뷰어 추천)
  5. 슬래시 커맨드 — 빠른 워크플로우 단축키

기본적으로 각 범주에서 1~2개의 최우선 추천을 보여준다. 특정 범주에 대해 더 많은 옵션을 원한다면 "MCP 서버를 더 추천해줘" 같이 요청하면 3~5개까지 확장해준다.

설치 방법

Claude Code 내에서 단 한 줄로 설치할 수 있다:

/plugin install claude-code-setup@claude-plugins-official

또는 플러그인 탐색 인터페이스를 통해 설치할 수도 있다:

/plugin > Discover

설치 후 플러그인을 사용하는 방법은 간단하다. Claude Code에서 다음과 같이 프롬프트를 입력하면 된다:

  • "이 프로젝트에 맞는 자동화를 추천해줘"
  • "Claude Code 설정을 도와줘"
  • "어떤 훅을 써야 할까?"
  • "MCP 서버는 무엇을 써야 해?"

실제 어떻게 작동하나 — 예시

이력서 파서(resume parser) 프로젝트를 예로 들어보자. 프로젝트에는 Python 백엔드, React 프론트엔드, PostgreSQL 데이터베이스가 있다.

claude-code-setup을 실행하면 대략 다음과 같은 추천을 받게 된다:

MCP 서버:

  • Playwright MCP — React UI 테스트 자동화용
  • PostgreSQL MCP — 데이터베이스 쿼리 직접 실행용

스킬:

  • security-reviewer — 개인정보 처리 코드 리뷰용
  • code-review — PR 리뷰 자동화용

훅:

  • 커밋 전 Python 린팅 (ruff, mypy)
  • 테스트 자동 실행

모두 프로젝트의 실제 기술 스택을 기반으로 한 맞춤 추천이다. 제네릭한 "모든 사람을 위한" 가이드가 아니다.

Claude Code 플러그인 생태계 전체 그림

claude-code-setup을 이해하려면 Claude Code 플러그인 시스템 전체를 이해하는 것이 도움이 된다.

Anthropic은 공식 플러그인 디렉토리를 GitHub에 공개했다 (anthropics/claude-plugins-official). 20,000개 이상의 스타를 받은 이 저장소는 두 가지로 구성된다:

  • /plugins — Anthropic이 직접 개발하고 유지하는 공식 플러그인
  • /external_plugins — 파트너사와 커뮤니티의 서드파티 플러그인

공식 플러그인 목록

현재 공식 디렉토리에는 다음과 같은 플러그인들이 포함되어 있다:

LSP(Language Server Protocol) 플러그인:

  • typescript-lsp, pyright-lsp, rust-analyzer-lsp, gopls-lsp
  • clangd-lsp, csharp-lsp, jdtls-lsp, kotlin-lsp
  • lua-lsp, php-lsp, ruby-lsp, swift-lsp

개발 워크플로우:

  • code-review — AI 코드 리뷰
  • feature-dev — 기능 개발 워크플로우
  • code-modernization — 레거시 코드 현대화
  • commit-commands — 커밋 자동화
  • pr-review-toolkit — PR 리뷰 툴킷

Claude Code 설정 및 플러그인 개발:

  • claude-code-setup — 바로 이 플러그인
  • claude-md-management — CLAUDE.md 관리
  • plugin-dev — 플러그인 개발 도우미
  • skill-creator — 커스텀 스킬 제작

외부 플러그인 (파트너사):

  • github, gitlab, linear, asana
  • firebase, playwright, terraform
  • context7, discord, telegram

플러그인 구조

각 플러그인은 표준 구조를 따른다:

plugin-name/
├── .claude-plugin/
│   └── plugin.json      # 플러그인 메타데이터 (필수)
├── .mcp.json            # MCP 서버 설정 (선택)
├── commands/            # 슬래시 커맨드 (선택)
├── agents/              # 에이전트 정의 (선택)
├── skills/              # 스킬 정의 (선택)
└── README.md            # 문서

두 가지 마켓플레이스: 공식 vs 커뮤니티

Claude Code 문서에 따르면 Anthropic은 두 개의 공개 플러그인 마켓플레이스를 운영한다:

  • claude-plugins-official — Anthropic이 큐레이션하는 공식 마켓플레이스. Claude Code에 기본으로 포함된다.
  • claude-community — 서드파티 제출을 검토하는 커뮤니티 마켓플레이스.

공식 마켓플레이스는 별도의 공개 신청 절차 없이 Anthropic이 직접 큐레이션한다. 커뮤니티 마켓플레이스는 제출 폼을 통해 외부 기여자들이 참여할 수 있다.

왜 플러그인 시스템이 중요한가

이전에는 Claude Code의 많은 커스터마이징이 프로젝트 `.claude/` 디렉토리 안에 있었다 — 커스텀 커맨드, 에이전트, 스킬, 훅 등. 이 방식은 개인 워크플로우나 단일 프로젝트에는 괜찮지만, 여러 프로젝트에 걸쳐 재사용하거나 팀에 일관되게 배포하기엔 한계가 있었다.

플러그인 시스템이 이 문제를 해결한다:

  • 동일한 설정을 여러 프로젝트에 설치할 수 있다
  • 커맨드와 스킬에 네임스페이스가 적용되어 충돌이 줄어든다
  • 마켓플레이스를 통해 배포 및 업데이트가 가능하다
  • 팀이 내부 베스트 프랙티스를 표준 플러그인으로 패키징할 수 있다
  • 커뮤니티가 특정 프레임워크, 언어, SaaS 서비스를 위한 확장을 만들 수 있다

VS Code 익스텐션, JetBrains 플러그인, 브라우저 확장과 비슷한 구조다. 안정적인 플러그인 생태계가 생기면 Claude Code는 단순한 CLI 도구가 아닌 플랫폼이 된다.

보안 주의사항

Anthropic의 공식 README는 중요한 경고로 시작한다:

플러그인을 설치, 업데이트, 또는 사용하기 전에 반드시 신뢰할 수 있는지 확인하라. Anthropic은 플러그인에 포함된 MCP 서버, 파일, 기타 소프트웨어를 통제하지 않으며 의도대로 작동하거나 변경되지 않을 것을 보장할 수 없다.

실용적인 보안 체크리스트:

  • 설치 전 플러그인 홈페이지와 README를 읽어라
  • .mcp.json, 훅, 실행 스크립트, 백그라운드 모니터가 포함되어 있는지 확인하라
  • 계정, 코드 저장소, 채팅 도구, 클라우드 서비스에 접근하는 플러그인은 각별히 주의하라
  • 중요한 프로젝트에 활성화하기 전에 샌드박스나 테스트 저장소에서 먼저 테스트하라
  • 팀 환경에서는 플러그인 출처와 버전을 중앙에서 검토하라

AI 코딩 플러그인은 일반적인 에디터 테마보다 훨씬 높은 권한을 가진다. 프로젝트 파일 읽기, 외부 서비스 호출, 로컬 명령 실행, 커밋 및 배포 흐름에 영향을 줄 수 있다. "작은 도구를 설치하는 것"보다 훨씬 엄격한 신뢰 기준을 적용해야 한다.

시작하는 방법

Claude Code를 이미 설치했지만 제대로 활용하지 못하고 있다면, 지금 당장 할 수 있는 한 가지:

/plugin install claude-code-setup@claude-plugins-official

설치 후 Claude Code에서 물어보자:

이 프로젝트에 맞는 Claude Code 자동화를 추천해줘

플러그인이 코드베이스를 분석하고, 프로젝트에 실제로 유용한 것들을 추천해줄 것이다. 모든 것을 적용할 필요는 없다 — 1~2가지만 먼저 시도해보고, 실제로 워크플로우를 개선하는지 확인해보자.

Claude Code는 혼란스러운 도구가 아니다. 제대로 설정되지 않은 도구일 뿐이었다. 이제 그 설정을 도와주는 공식 플러그인이 있다.
