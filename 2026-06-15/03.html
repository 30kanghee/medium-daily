20 Most Important AI Concepts Explained in Just 20 Minutes

Deep concept · Let's Code Future · 22분 · 원문 · freedium

LLM부터 에이전틱 AI까지, 현대 AI 시스템을 구성하는 20가지 핵심 개념을 전문 용어 없이 명쾌하게 정리했다.

요약

AI는 빠르게 발전하지만 기본 개념은 비교적 소수의 핵심 아이디어로 구성되어 있다. 이 글은 LLM, 프롬프트, 임베딩, 벡터 데이터베이스, RAG, 파인튜닝, 함수 호출, AI 에이전트, MCP 등 20가지 핵심 개념을 입문자도 이해할 수 있도록 명확하게 설명한다. 각 개념이 어떻게 서로 연결되어 하나의 흐름을 이루는지 보여주며, 이 흐름을 이해하면 대부분의 현대 AI 시스템 구조가 한눈에 보인다. 저자는 프레임워크보다 개념 자체를 먼저 익힐 것을 권고하며, AI의 미래는 그것에 대해 말하는 사람이 아니라 실제로 작동 원리를 이해하는 사람에게 속한다고 강조한다.

인공지능은 너무 빠르게 움직여서 많은 개발자들이 시작도 전에 압도당한다.

매주 새로운 용어들이 등장한다:

  • 에이전트 (Agents)
  • RAG
  • 파인튜닝 (Fine-Tuning)
  • 임베딩 (Embeddings)
  • MCP
  • 벡터 데이터베이스 (Vector Databases)
  • 함수 호출 (Function Calling)

갑자기 모든 사람이 당신이 이해하지 못하는 언어로 말하는 것처럼 느껴진다.

좋은 소식이 있다. 기본 개념을 이해하는 데 몇 달의 공부가 필요하지 않다.

이 20가지 개념을 배우면, 대부분의 현대 AI 애플리케이션이 어떻게 구축되는지 이해하게 될 것이다.

시작하자.

1. LLM (대규모 언어 모델)

현대 AI 시스템의 두뇌다.

예시:

  • ChatGPT
  • Claude
  • Gemini
  • Llama

LLM은 맥락을 기반으로 다음 토큰(단어 조각)을 예측한다.

모든 것이 여기서 시작된다.

2. 프롬프트 (Prompt)

AI에게 주는 지시다.

예시:

FastAPI를 사용해서 REST API를 만들어줘.

더 나은 프롬프트는 보통 더 나은 결과를 만들어낸다.

3. 프롬프트 엔지니어링 (Prompt Engineering)

더 나은 결과를 생성하는 프롬프트를 설계하는 기술이다.

포함 요소:

  • 맥락 (Context)
  • 제약 조건 (Constraints)
  • 예시 (Examples)
  • 출력 형식 지정 (Output formatting)

여전히 AI 분야에서 가장 높은 레버리지를 가진 기술 중 하나다.

4. 토큰 (Tokens)

AI 모델은 단어를 보지 않는다.

토큰을 본다.

토큰은 처리와 과금에 사용되는 텍스트의 작은 조각이다.

토큰을 더 많이 사용할수록 비용과 컨텍스트 소비가 늘어난다.

5. 컨텍스트 윈도우 (Context Window)

모델이 대화 중에 기억할 수 있는 정보의 양이다.

AI의 단기 기억이라고 생각하면 된다.

컨텍스트 윈도우가 클수록 더 복잡한 워크플로우가 가능하다.

6. 임베딩 (Embeddings)

임베딩은 텍스트를 숫자 벡터로 변환한다.

이를 통해 AI 시스템이 의미적 뜻을 이해할 수 있다.

예를 들어:

"자동차"

와

"승용차"

는 비슷한 임베딩을 생성한다.

7. 벡터 데이터베이스 (Vector Database)

임베딩을 저장하기에 최적화된 데이터베이스다.

인기 있는 예시:

  • Pinecone
  • Weaviate
  • Chroma
  • Qdrant

이것이 시맨틱 검색 시스템을 구동한다.

8. RAG (검색 증강 생성, Retrieval-Augmented Generation)

AI에서 가장 중요한 개념 중 하나다.

RAG는 모델이 응답을 생성하기 전에 외부 정보를 검색할 수 있게 한다.

학습 데이터에만 의존하는 대신, 모델이 다음에 접근할 수 있다:

  • PDF
  • 문서
  • 데이터베이스
  • 내부 지식

대부분의 기업용 AI 시스템은 RAG를 사용한다.

9. 파인튜닝 (Fine-Tuning)

특화된 데이터로 기존 모델을 추가 학습하는 것이다.

다음이 필요할 때 유용하다:

  • 도메인 전문성
  • 일관된 행동
  • 산업별 지식

모든 AI 애플리케이션에 파인튜닝이 필요한 것은 아니다.

10. 함수 호출 (Function Calling)

AI 모델이 소프트웨어 함수를 트리거할 수 있게 한다.

예시:

사용자가 물어본다:

날씨가 어때?

AI가 날씨 API를 호출한다.

실시간 데이터를 반환한다.

이것이 AI가 외부 시스템과 상호작용하는 방식이다.

11. 도구 (Tools)

도구는 AI에게 대화를 넘어서는 능력을 준다.

예시:

  • 검색
  • 이메일
  • 데이터베이스 접근
  • 계산기
  • API

도구 없이 AI는 말할 수만 있다. 도구가 있으면 AI는 일할 수 있다.

12. AI 에이전트 (AI Agents)

AI 에이전트는 단순히 질문에 답하지 않는다.

다음을 할 수 있다:

  • 계획 수립
  • 추론
  • 도구 사용
  • 작업 실행

에이전트를 LLM으로 구동되는 자율 작업자로 생각하면 된다.

13. 멀티 에이전트 시스템 (Multi-Agent Systems)

여러 에이전트가 함께 작동하는 시스템이다.

예시:

리서치 에이전트 → 분석 에이전트 → 작성 에이전트

각 에이전트가 서로 다른 작업을 전문으로 한다.

14. MCP (모델 컨텍스트 프로토콜, Model Context Protocol)

AI 모델이 도구 및 데이터 소스와 연결하는 표준화된 방식이다.

MCP를 AI의 USB-C라고 생각하면 된다.

하나의 프로토콜. 수많은 통합.

15. 환각 (Hallucinations)

AI가 자신 있게 잘못된 정보를 생성하는 현상이다.

모든 모델은 가끔 환각을 일으킨다.

그래서 검증이 중요하다. 항상 중요한 정보를 확인하라.

16. 온도 (Temperature)

창의성을 제어한다.

낮은 온도:

  • 더 예측 가능함
  • 더 결정론적임

높은 온도:

  • 더 창의적임
  • 더 다양함

작업에 따라 유용하게 활용할 수 있다.

17. 추론 (Inference)

학습된 모델에서 출력을 생성하는 과정이다.

학습(Training)은 가르친다. 추론(Inference)은 수행한다.

대부분의 개발자는 주로 추론과 상호작용한다.

18. 에이전트 메모리 (Agent Memory)

AI 에이전트가 시간이 지남에 따라 정보를 기억할 수 있게 한다.

예시:

  • 사용자 선호도
  • 이전 대화
  • 진행 중인 작업

메모리는 더 개인화된 경험을 만든다.

19. AI 워크플로우 (AI Workflows)

AI 행동의 구조화된 시퀀스다.

예시:

입력 → 리서치 → 분석 → 작성 → 검토

많은 성공적인 AI 제품은 완전 자율 에이전트가 아닌 워크플로우다.

20. 에이전틱 AI (Agentic AI)

AI 시스템의 다음 진화다.

에이전틱 AI는 다음을 결합한다:

  • 추론 (Reasoning)
  • 계획 (Planning)
  • 도구 (Tools)
  • 메모리 (Memory)
  • 자율성 (Autonomy)

단순히 질문에 답하는 대신, 이 시스템들은 목표를 달성한다.

이것이 업계의 상당 부분이 향하는 방향이다.

이 개념들이 어떻게 연결되는가

간략한 전체 그림:

사용자 프롬프트
↓
LLM
↓
도구 + 함수 호출
↓
RAG + 벡터 데이터베이스
↓
메모리
↓
에이전트
↓
완료된 작업

이 흐름을 이해하면, 현대 AI 시스템을 이해하기가 훨씬 쉬워진다.

개발자가 먼저 배워야 할 것

오늘 시작한다면, 이 순서로 개념에 집중하라:

  1. LLM
  2. 프롬프트
  3. API
  4. 임베딩
  5. RAG
  6. 함수 호출
  7. 에이전트
  8. MCP

새로운 프레임워크를 쫓기 전에 이 기본 개념들을 마스터하라.

도구는 바뀔 것이다. 개념은 바뀌지 않는다.

마치며

AI는 수십 개의 연결되지 않은 유행어로 볼 때 믿을 수 없을 만큼 복잡해 보일 수 있다.

하지만 대부분의 현대 AI 시스템은 비교적 작은 핵심 아이디어들로 구축된다.

이 20가지 개념을 배워라.

어떻게 연결되는지 이해하라.

몇 가지 작은 프로젝트를 만들어 봐라.

그러면 온라인에서 AI에 대해 이야기하는 대다수의 사람들보다 AI에 대해 더 많이 알게 될 것이다.

왜냐하면 미래는 AI에 대해 말하는 사람들에게 속하지 않기 때문이다.

AI가 실제로 어떻게 작동하는지 이해하는 사람들에게 속한다.
