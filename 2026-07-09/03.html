8 Advanced Data Pipeline Patterns Used in High-Scale Systems

Saurav Singh · Towards Data Engineering · 10분 읽기 · 원문

카프카냐 스파크냐보다 중요한 건 대규모 데이터 파이프라인이 실전에서 버티게 해주는 8가지 설계 패턴이다.

요약

저자는 어떤 도구를 쓰느냐보다 어떤 아키텍처 패턴을 적용하느냐가 파이프라인의 생존을 좌우한다며 8가지 패턴을 소개한다. 실시간 처리와 배치 처리를 함께 운영하는 람다 아키텍처와, 이를 단순화해 스트림 처리 하나로 통합한 카파 아키텍처를 대비시키고, DB 갱신과 이벤트 발행을 하나의 트랜잭션으로 묶어 유실을 막는 아웃박스 패턴과 이를 뒷받침하는 체인지 데이터 캡처(CDC) 기법을 설명한다. 현재 상태 대신 변경 이력 전체를 저장해 감사 추적이 가능하게 하는 이벤트 소싱, 빠른 생산자가 느린 소비자를 압도하지 않도록 조절하는 백프레셔 기법도 다룬다. 마이크로서비스 간 분산 트랜잭션을 2단계 커밋 없이 처리하는 사가 패턴, 그리고 실패한 메시지를 무한정 재시도하거나 버리지 않고 별도로 모아 추적하는 데드레터 큐까지 소개하며 이 패턴들이 서로 맞물려야 진짜 안정적인 시스템이 된다고 강조한다. 결국 8가지를 한꺼번에 도입하는 게 목표가 아니라, 각 패턴이 어떤 문제를 해결하는지 이해해 필요할 때 꺼내 쓰는 것이 핵심이라는 게 글의 결론이다.
