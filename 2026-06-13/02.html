이번 주 ChatGPT, Cursor, Midjourney를 해지했다 — MacBook Pro M5 Max가 세 가지를 조용히 대체했다

Shreetej Ghodekar · Mac O'Clock · 10 min read · 원문 · freedium

36GB 통합 메모리를 갖춘 MacBook Pro M5 Max 하나로 Qwen 3.6 35B 모델을 로컬에서 구동해 ChatGPT·Cursor·Midjourney 구독을 모두 해지할 수 있다.

저자는 2년간 Anthropic·OpenAI·Midjourney에 매월 비용을 지불했지만, 2026년 들어 M5 Max 칩·Qwen 3.6 오픈소스 모델·성숙한 MLX 생태계가 동시에 갖춰지면서 구독을 전면 해지했다. 36GB 통합 메모리는 4비트 양자화된 35B 파라미터 모델을 완전히 오프라인으로 구동할 수 있는 최적의 스위트 스폿이며, LM Studio·Ollama·oMLX·mlx-lm 등 네 가지 경로로 모델을 실행할 수 있다. 에이전트 코딩(Cline, OpenHands), 이미지 생성(ComfyUI+Flux), 음성 전사(Whisper.cpp), RAG, LoRA 파인튜닝까지 모두 클라우드 없이 가능하다. 저자는 AI가 구독 서비스가 아닌 인프라로 전환되고 있다고 결론짓는다.

클라우드도, API 키도, 월정액도 없이 — 36GB 통합 메모리에서 Qwen 3.6(새로운 35B 에이전트 모델), ComfyUI, oMLX, 그리고 오픈소스 AI 스택 전체를 실행하는 2026년 실전 가이드

2년 동안 나는 Anthropic, OpenAI, Midjourney에 매달 비용을 지불하며 그것을 "비즈니스 비용"이라고 불렀다. 지난주에 나는 멈췄다. 모델이 나빠진 게 아니라 — 36GB 통합 메모리를 탑재한 14인치 MacBook Pro M5 Max가 충분히 좋아졌기 때문이다.

구체적으로, 세 가지가 동시에 이루어졌다:

  1. Apple이 M5 Max를 출시했다(2026년 3월). 32개의 GPU 코어 각각에 Neural Accelerator가 내장되어 전 세대 대비 LLM 프롬프트 처리 속도가 최대 4배 빨라졌다.
  2. Alibaba가 Qwen 3.6–35B-A3B를 오픈소스로 공개했다(2026년 4월 16일). 토큰당 3B 파라미터만 활성화하는 350억 파라미터 Mixture-of-Experts 에이전트 모델이다. Apache 2.0 라이선스. 노트북에 들어간다.
  3. MLX 생태계가 성숙해졌다. Ollama는 이제 MLX 기반으로 구축되었다. oMLX는 SSD 기반 KV 캐싱을 추가했다. LM Studio는 원클릭 설치 프로그램이 되었다. ComfyUI Desktop은 Apple Silicon용 베타 버전으로 출시되었다.

M5 Max(또는 솔직히 32GB+ 메모리를 갖춘 M 시리즈 칩)를 보유하고 있다면, 12개월 전에는 존재하지 않았던 개인 AI 워크스테이션을 갖게 된 것이다. 이 가이드는 에이전트 코딩, 이미지 생성, 음성, 파인튜닝 등 전체 스택을 오후 시간 안에 설치하고 작동시키는 방법을 다룬다.

이 노트북에서 로컬로 실제로 할 수 있는 것

설치 단계에 앞서, 36GB M5 Max가 잠금 해제하는 기능 목록을 소개한다. 아래 모든 기능은 설치 후 네트워크 연결 없이 실행된다:

  • 에이전트 코딩 — Qwen 3.6이 Cline, OpenHands, Aider, 또는 Claude Code(로컬 서버를 가리키도록 설정)를 구동한다. 계획 수립, 파일 편집, 테스트 실행, 반복 작업을 수행한다.
  • 비전 및 멀티모달 추론 — Qwen 3.6–35B-A3B에는 비전 인코더가 탑재되어 있어 스크린샷, PDF, UI 목업을 붙여 넣을 수 있다.
  • 이미지 생성 — ComfyUI Desktop에서 Flux, SDXL, Z-Image-Turbo 실행. M5 Max에서 1024×1024 이미지를 약 25~35초 만에 생성(Mac Mini M4 Pro는 50초, M5 Max는 코어별 Neural Accelerator 덕분에 실질적으로 더 빠름).
  • 음성 및 오디오 — Whisper.cpp가 Metal에서 수 시간 분량의 오디오를 수 분 만에 전사한다. 로컬 TTS와 페어링하면 완전한 음성 입출력이 가능하다.
  • 완전 오프라인 문서 RAG — Qwen과 함께 임베딩 모델과 리랭커를 실행해 프라이빗 지식 베이스를 구축한다. oMLX는 하나의 프로세스에서 세 가지 모델 유형 모두를 서빙할 수 있다.
  • 파인튜닝 — mlx-lm은 LoRA와 QLoRA를 기본 지원한다. 이 노트북에서 본인의 데이터로 7~9B 모델을 파인튜닝할 수 있다.
  • MCP 도구 호출 — Qwen 3.6의 도구 호출 형식은 Model Context Protocol과 호환되므로, 편집기와 동일한 표준을 통해 파일 시스템, 웹, 셸 접근 권한을 부여할 수 있다.

이제 세팅해보자.

36GB M5 Max가 최적의 스위트 스폿인 이유

M5 Max를 탑재한 14인치 MacBook Pro는 기본 구성으로 18코어 CPU, 32코어 GPU, 36GB 통합 메모리를 제공한다. Apple은 M1 세대 대비 LLM 프롬프트 처리 속도 최대 4배, AI 이미지 생성 속도 최대 8배 향상을 주장한다.

AI 작업에서 중요한 세부 사항은 코어 수가 아니라 통합 메모리 아키텍처다. Mac에서는 CPU, GPU, Neural Engine이 모두 동일한 메모리 풀을 공유한다. PCIe 버스를 통해 웨이트를 이동시킬 필요가 없다. 4비트 양자화된 Qwen 3.6–35B-A3B는 KV 캐시에 따라 약 19~22GB를 차지하며, ComfyUI, IDE, 브라우저, Slack을 위해 14~17GB가 남는다.

이를 개별 GPU 노트북과 비교하면: 이 모델을 시도하려면 24GB RTX 4090 모바일이 필요하고, 비용은 더 많이 들며, 더 뜨겁게 구동되고, 무게는 두 배가 된다.

파트 1 — Qwen 3.6 실행하기 (경로 선택)

오늘날 Apple Silicon에서 Qwen 3.6을 서빙하는 네 가지 진지한 방법이 있다. 난이도/성능 스펙트럼에 위치한다. 터미널 시간을 얼마나 즐기느냐에 따라 선택하라.

경로 A — LM Studio (5분, 터미널 없음)

LM Studio는 가장 부드러운 진입점이다. GGUF와 MLX 백엔드를 동일한 창에서 실행하는 Mac 앱이며, Apple Silicon에서는 MLX를 인퍼런스 백엔드로 사용하여 네이티브 Metal GPU 가속을 지원한다.

  1. lmstudio.ai에서 다운로드 → .dmg 설치.
  2. LM Studio 열기 → 사이드바에서 Discover 클릭 → qwen3.6 검색.
  3. 여러 양자화 변형이 표시된다. Unsloth의 Q4_K_S(~20.9GB)를 선택하라 — 36GB Mac에서 품질/메모리 최적점이다. 품질을 조금 더 원한다면 Q4_K_XL도 들어간다.
  4. 다운로드 아이콘 클릭. 커피 한 잔 마시자.
  5. 다운로드 완료 후 Load 클릭. Activity Monitor를 확인하면 약 20GB의 메모리 압력이 표시된다.
  6. Local Server 탭 클릭 → Start Server. 이제 http://localhost:1234/v1에 OpenAI 호환 엔드포인트가 생긴다.

끝이다. LM Studio UI에서 채팅하거나, 로컬 URL에 OpenAI 호환 클라이언트를 연결하면 된다.

경로 B — Ollama (CLI 친숙도)

Apple Silicon의 Ollama는 버전 0.18~0.19부터 MLX 기반으로 구축되었으며, M5 칩에서는 GPU Neural Accelerator를 직접 활용한다. Ollama 0.19 벤치마크는 int4 양자화로 Qwen 실행 시 프리필 1,851 토큰/초, 디코딩 134 토큰/초를 보여준다.

# Homebrew로 설치
brew install ollama
# 서버 시작 (또는 메뉴바 앱 사용)
ollama serve
# Qwen 3.6 MLX 빌드 다운로드 — 약 22GB
ollama pull qwen3.6:35b-mlx
# 채팅
ollama run qwen3.6:35b-mlx

Ollama는 자동으로 http://localhost:11434/v1에 OpenAI 호환 API를 노출한다. 완료.

경로 C — oMLX (파워 유저의 선택)

oMLX는 Apple Silicon을 위해 특별히 구축된 네이티브 macOS LLM 인퍼런스 서버다. 헤드라인 기능은 2계층 KV 캐시다 — SSD 콜드 캐시로 백업된 RAM 핫 캐시를 통해, 반복적인 시스템 프롬프트나 공유 코드베이스 컨텍스트가 돌아올 때 재계산 대신 디스크에서 복원한다. 이를 통해 Claude Code 세션 같은 작업에서 에이전트의 첫 토큰 생성 시간이 30~90초에서 1~3초로 단축된다.

연속 배칭, 페이지드 프리픽스 공유, OpenAI 및 Anthropic API 엔드포인트를 지원하며, LLM/VLM/임베딩/리랭커 모델을 동시에 실행한다.

# 설치
brew install jundot/tap/omlx
# 백그라운드 서비스로 실행
brew services start omlx

그런 다음 oMLX 메뉴바 아이콘 클릭 → http://localhost:8000/admin에서 관리자 대시보드 열기 → HuggingFace에서 Qwen3.6-35B-A3B-4bit 검색 → 원클릭 다운로드. OpenAI 호환 엔드포인트는 http://localhost:8000/v1이며, Claude Code 같은 도구를 위한 Anthropic 호환 /v1/messages도 있다.

Qwen 3 coder 30B-A3B 및 Qwen 3.6 35B-A3B에 대한 oMLX 벤치마크.

하루 종일 에이전트 코딩을 할 예정이라면 oMLX를 사용하라 — SSD KV 캐시는 첫 시간 내에 그 가치를 입증한다.

경로 D — Raw mlx-lm (스크립트 및 파인튜닝용)

스크립트를 작성하거나 결국 파인튜닝을 하려면 Apple의 공식 mlx-lm을 설치하라:

pip install mlx-lm
mlx_lm.generate \
  --model mlx-community/Qwen3.6-35B-A3B-4bit \
  --max-tokens 500 \
  --temp 0.7 \
  --prompt "Write a Python function that deduplicates a list while preserving order."

오버헤드 없는 서버를 원할 때 시작하는 방법도 있다:

mlx_lm.server --model mlx-community/Qwen3.6-35B-A3B-4bit --port 8080

솔직한 추천: 작동하는지 확인하기 위해 LM Studio로 시작한 다음, 익숙해지면 oMLX로 전환하라. mlx-lm은 스크립트를 작성하거나 파인튜닝할 때만 사용하라.

파트 2 — Qwen 3.6을 코드를 작성하는 에이전트로 만들기

모델은 그저 신탁에 불과하다. 에이전트의 마법 — 파일 편집, 터미널 명령, 루프 실행 테스트 — 을 얻으려면 하네스가 필요하다. 좋은 소식: 모든 현대적인 코딩 에이전트는 OpenAI의 API를 사용하고, 로컬 서버도 OpenAI의 API를 사용한다.

옵션 1: Cline (VS Code, 가장 쉬움)

  1. VS Code 확장 마켓플레이스에서 Cline을 설치한다.
  2. Cline 설정 열기 → API Provider → "OpenAI Compatible" 선택.
  3. Base URL: http://localhost:1234/v1(LM Studio) 또는 http://localhost:8000/v1(oMLX) 또는 http://localhost:11434/v1(Ollama).
  4. Model ID: qwen3.6:35b-mlx(서버가 보고하는 값 사용 — LM Studio는 Local Server 탭에 표시).
  5. Cline 채팅 패널을 열고 "/health 엔드포인트가 있는 Python FastAPI 서버 만들기"라고 입력한다. 파일 편집, 명령 실행, 반복 작업이 이루어지는 것을 지켜보라.

옵션 2: OpenHands (완전한 에이전트 루프)

OpenHands는 에이전트 코딩 최적화와 대형 컨텍스트 윈도우를 이유로 Qwen 3.6–35B-A3B를 첫 번째로 시도할 로컬 모델로 명시적으로 추천한다. 로컬 LLM 가이드를 따라 LM Studio 또는 oMLX에 연결하면 전체 저장소를 작성할 수 있는 샌드박스 에이전트를 갖게 된다.

옵션 3: 로컬 백엔드를 사용한 Claude Code

이것은 약간 기묘하지만 매우 재미있다. oMLX는 Anthropic 호환 /v1/messages 엔드포인트를 노출하므로, Claude Code가 로컬 모델을 가리키도록 설정할 수 있다. ANTHROPIC_BASE_URL=http://localhost:8000과 ANTHROPIC_API_KEY=anything을 설정하고 Claude Code를 실행하면, Anthropic과 대화한다고 믿으면서 Qwen 3.6과 통신한다. 동일한 에이전트 UX, 데이터 외부 전송 없음.

알아둘 가치 있는 주의 사항

Qwen 3.6은 Anthropic 도구 사용 포맷이 아닌 OpenAI 스타일의 function_call 스키마를 사용한다. 대부분의 현대적인 하네스(OpenHands, Cline, Aider)는 둘 다 처리한다. 직접 하네스를 만든다면 소형 어댑터 계층이 필요하다.

파트 3 — 이미지 생성을 위한 ComfyUI (Midjourney 대체제)

ComfyUI는 전문가들이 사용하는 노드 기반 이미지 생성 인터페이스다. AI 파이프라인의 Photoshop이라고 생각하라 — 원하는 정확한 워크플로우를 구축하기 위해 노드를 연결한다. 공식 ComfyUI Desktop은 이제 Apple Silicon만 지원하는 macOS 베타 버전이 있다.

ComfyUI Desktop 설치 (베타)

  1. docs.comfy.org로 이동 → macOS 설치 프로그램 다운로드.
  2. 패키지를 더블클릭한다. MPS 지원을 갖춘 Python과 PyTorch가 번들로 제공되므로 환경을 관리할 필요가 없다.
  3. ComfyUI Desktop을 실행한다. 첫 실행 시 작업 공간 폴더(보통 ~/Documents/ComfyUI)가 설정된다.

모델 다운로드

ComfyUI는 모델이 포함되어 있지 않으므로 직접 가져와야 한다. M5 Max에서 시작할 세 가지 모델을 추천한다:

  • Flux.1 [dev] — Black Forest Labs의 플래그십. 사실적이며 프롬프트 준수도가 우수하다. 전체 정밀도에서 ~24GB이지만, GGUF 양자화 빌드로 ~12GB까지 줄일 수 있다. .safetensors를 models/unet/에 넣거나 양자화 버전에는 GGUF 로더를 사용하라.
  • SDXL 1.0 — 믿을 수 있는 워크호스. Flux보다 빠르며 스타일리시한 출력에 적합하다. models/checkpoints/에 배치.
  • Z-Image-Turbo — Flux의 약 10분의 1 규모로 가볍지만 크기 대비 놀라운 품질. M5 Max에서 1024×1024 이미지를 약 25~30초 만에 생성한다.

첫 번째 워크플로우

ComfyUI에서 Template 메뉴 → Image Generation → Flux Dev 또는 SDXL 선택. Queue Prompt 클릭. 완료. 1024×1024 Flux 생성은 약 25~35초 안에 완료된다; M4 Pro Mac Mini는 동일 작업에 ~50초가 소요되며, M5 Max는 코어별 Neural Accelerator 덕분에 의미 있게 더 빠르다.

메모리 팁

macOS의 ComfyUI는 GPU 가속에 Metal Performance Shaders(MPS)를 사용한다. Qwen 3.6과 ComfyUI를 동시에 로드했을 때 메모리 압박이 발생하면, ComfyUI 실행 전에 셸에서 다음을 설정하라:

export PYTORCH_MPS_HIGH_WATERMARK_RATIO=0.0

이는 PyTorch에게 메모리를 더 적극적으로 공유하도록 지시한다. 36GB에서는 Qwen 3.6 4비트(~20GB)와 Z-Image-Turbo(~6GB)를 동시에 로드할 수 있어야 한다. Flux는 더 빡빡하다 — 전체 정밀도 Flux를 원한다면 Qwen을 닫아라.

파트 4 — 오픈소스 AI 스택의 나머지

Qwen과 ComfyUI가 작동하면 다음 항목들을 추가할 수 있다. 이 중 어느 것도 비용이 들지 않는다.

Whisper.cpp — 음성 전사

Apple의 Metal 백엔드는 M 시리즈에서 Whisper.cpp를 매우 빠르게 만들었다. 1시간 분량의 오디오를 1분 이내에 전사한다.

brew install whisper-cpp
whisper-cpp -m models/ggml-large-v3.bin -f meeting.wav

임베딩 + 리랭킹

RAG를 위해서는 소형 임베딩 모델과 리랭커가 필요하다. oMLX는 Qwen과 함께 두 가지를 동시에 서빙한다 — bge-m3와 bge-reranker-v2-m3를 ~/.omlx/models/ 폴더에 넣으면 자동 검색된다. LlamaIndex 같은 도구를 세 개의 엔드포인트 모두에 연결하면 된다.

mlx-lm LoRA로 파인튜닝

사람들이 간과하는 킬러 기능이다. M5 Max에서 도메인 데이터셋으로 7~9B 모델을 몇 시간 만에 파인튜닝할 수 있다:

mlx_lm.lora \
  --model mlx-community/Qwen2.5-7B-Instruct-4bit \
  --train \
  --data ./my_data \
  --iters 1000 \
  --batch-size 2

36GB Mac은 Qwen 3.6 전체 35B를 파인튜닝할 여유가 없지만, 도메인 특화 7B 모델의 경우 실제로 사용 가능하다.

Open WebUI — 위의 모든 것을 위한 ChatGPT 스타일 UI

brew install open-webui
open-webui serve

http://localhost:8080으로 접속하고, LM Studio / Ollama / oMLX 엔드포인트에 연결하면 히스토리, RAG, 문서 업로드, 멀티모델 채팅을 갖춘 로컬 스택용 ChatGPT 스타일 인터페이스를 얻게 된다.

실제 성능은 어떤 느낌인가

36GB를 갖춘 14인치 M5 Max에서의 실제 수치:

  • Qwen 3.6–35B-A3B (MLX 4비트): 에이전트 작업에서 약 90~130 토큰/초 디코딩. 모델을 기다리는 것이 아니라 읽는 속도를 따라가는 것이 더 어렵다.
  • oMLX SSD 캐시로 첫 토큰까지 시간: 캐시된 프리픽스에서 1~3초 대 콜드 상태 30~90초.
  • Flux 1024×1024: ~25~35초. Z-Image-Turbo: ~25~30초. SDXL: ~10~15초.
  • 배터리 영향: 상당하다. 지속적인 에이전트 실행 시 충전을 계획하라. 대기 상태에서는 정상이다.

Claude Opus 4.7만큼 똑똑한가? 아니다 — Opus 4.7은 여전히 SWE-bench에서 약 84%로 최상위권을 차지한다. 그러나 Qwen 3.6–35B-A3B는 일상적인 코딩에서 충분히 경쟁적이어서 일주일 동안 유료 모델을 찾지 않았다. 그리고 35B 에이전트가 비행기 모드로 이동 중인 노트북에서 자율적으로 버그를 수정하는 것을 처음 봤을 때, 수치는 더 이상 중요하지 않았다.

이것이 바꾸는 것

내가 계속 스스로에게 하는 이야기는 AI가 구독 서비스가 아닌 인프라가 되고 있다는 것이다. 2년 동안 유일한 실행 가능한 워크플로우는 토큰당 다른 사람의 GPU를 임대하는 것이었다. 그 모델은 훌륭한 제품을 만들어냈다 — 하지만 Cmd-K를 누를 때마다 코드, 프롬프트, 데이터가 데이터 센터를 거쳐간다는 의미이기도 했다.

M5 Max 실리콘, MLX 생태계, Qwen 3.6 같은 Apache 라이선스 에이전트 모델의 조합은 노트북 로컬 인퍼런스가 실제로 경쟁력 있는 첫 번째 시대다 — 장난감이 아닌 일상적인 도구로서.

M5 Max 업그레이드를 망설이고 있다면, 이것이 투자를 회수하는 사용 사례다. 이미 있다면, 구입 당시에는 존재하지 않았던 AI 워크스테이션을 갖고 있는 것이다. 어느 쪽이든, 오늘 밤 LM Studio를 설치하고, Qwen 3.6을 다운로드하고, 직접 확인해 보라.

클라우드는 훌륭하다. 스택을 직접 소유하는 것이 더 낫다.

도움이 되었다면 박수를 치고 팔로우하라 — 매주 로컬 AI에 대한 실습 심층 분석을 게시하고 있다. 설정 질문이 있거나 특정 항목을 벤치마크해 주길 원한다면 댓글을 남겨라.
