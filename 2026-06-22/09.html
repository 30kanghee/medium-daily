I Used the Terminal Wrong for Years

Tushar Kanjariya · 6분 읽기 · 원문 · freedium

터미널을 단순 도구가 아닌 환경으로 대하는 순간, 시니어 개발자처럼 흐르기 시작한다.

대부분의 개발자는 첫 튜토리얼에서 배운 방식 그대로 터미널을 사용한다. 명령어를 치고, 엔터를 누르고, 반복하는 방식이다. 그것도 작동하지만 느리고, 작은 지연이 쌓이면 실제 마찰이 된다. 이 글은 sudo !!, Ctrl+R 리버스 검색, 타이포 즉시 수정, 별칭(alias), 키보드 단축키, tmux, fzf 등 시니어 개발자가 매일 실제로 쓰는 터미널 트릭을 소개한다. 이 트릭들은 설정하는 데 10분도 걸리지 않지만 수개월, 수년에 걸쳐 작업을 훨씬 부드럽게 만드는 복리 효과를 가져온다.

나는 시니어 개발자가 배포 문제를 90초 안에 고치는 것을 지켜봤다.

구글링도 없었다. 치트시트에서 복붙도 없었다. 그냥 터미널과 대화하듯 자연스럽게 흘러갔다.

나는 그가 뭔가 비밀 툴셋을 가진 줄 알았다. 아니었다. 그는 그냥 나보다 터미널을 더 잘 알고 있었을 뿐이다.

대부분의 개발자가 가진 문제

대부분의 개발자는 첫 튜토리얼에서 배운 방식 그대로 터미널을 사용한다. 명령어를 치고, 엔터를 누르고, 반복한다.

그것도 작동한다.

하지만 느리다.

그리고 시간이 지나면서 그 작은 지연들이 실제 마찰로 쌓인다.

내가 깨닫지 못했던 것이 있다:

터미널은 단순한 도구가 아니다. 그것은 환경이다.

그렇게 대하기 시작하면 모든 것이 달라진다.

트릭 #1: 이미 실행한 명령을 다시 치지 마라

이건 배웠을 때 아팠다.

긴 명령을 실행했다. 권한 오류로 실패했다. 이제 sudo로 재시도하고 싶다. 위 화살표를 눌러 커서를 맨 앞으로 이동하는 대신 그냥 이렇게 하면 된다:

sudo !!

!!는 마지막 명령어 전체로 확장된다. 그게 전부다. 전부 다시 치는 대신 두 글자면 된다.

이건 훨씬 더 좋다:

mkdir my-project
cd !$

!$는 이전 명령의 마지막 인수를 준다. 폴더를 만들고 이름을 두 번 치지 않고 바로 그 안으로 들어갔다. 나는 !$를 하루에 20번은 쓴다. 작지만 제거하는 마찰이 진짜다.

트릭 #2: Ctrl+R은 터미널의 검색 엔진이다

누구나 위 화살표가 히스토리를 순환한다는 걸 안다. 하지만 그건 선형적이다. 원하는 것을 찾을 때까지 계속 눌러야 한다.

Ctrl+R은 전체 히스토리에서 역방향 검색을 한다. 명령의 일부를 타이핑하기 시작하면 가장 최근 일치를 찾아준다.

(reverse-i-search)`docker': docker-compose up -d --build

다시 Ctrl+R을 눌러 더 오래된 일치를 순환한다. Enter로 실행한다. Esc로 먼저 수정한다.

보너스: 더 멀리 검색할 수 있도록 히스토리 크기를 늘려라. .bashrc 또는 .zshrc에 추가:

HISTSIZE=10000
HISTFILESIZE=20000
HISTCONTROL=ignoredups:erasedups

이제 히스토리가 10,000개 명령까지 거슬러 올라가고 중복으로 어수선해지지 않는다.

트릭 #3: 오타를 즉시 수정하기

방금 오타가 있는 명령을 실행했다. 전형적인 예:

git chekcout main

위를 눌러 수동으로 수정하는 대신 ^old^new 치환 트릭을 사용하라:

^chekcout^checkout

마지막 명령에서 chekcout을 checkout으로 교체하고 즉시 실행한다.

팀원들이 멈추고 물어볼 트릭이다. "잠깐, 어떻게 한 거야?" 설명할 때 기분이 좋을 것이다.

트릭 #4: 별칭은 무료 생산성이다 — 활용하라

일주일에 다섯 번 이상 타이핑하는 모든 명령은 별칭이 필요하다. 별칭은 ~/.bashrc(Linux) 또는 ~/.zshrc(macOS)에 산다. 실제로 내가 쓰는 스타터 팩:

# 탐색
alias ..='cd ..'
alias ...='cd ../..'
alias ~='cd ~'
alias -- -='cd -'          # 이전 디렉토리로 돌아가기

# 목록 표시
alias ll='ls -alF'
alias lt='ls -ltr'         # 시간순으로 목록, 최신이 마지막

# 안전망 (덮어쓰기 전 확인)
alias rm='rm -i'
alias cp='cp -i'
alias mv='mv -i'

# Git 단축키
alias gs='git status'
alias ga='git add .'
alias gc='git commit -m'
alias gp='git push'
alias gl='git log --oneline --graph --decorate'
alias gco='git checkout'

# Docker 단축키
alias dps='docker ps'
alias dc='docker-compose'
alias dcu='docker-compose up -d'
alias dcd='docker-compose down'

# 개발 단축키
alias ni='npm install'
alias nrd='npm run dev'
alias nrb='npm run build'

# 편집 후 쉘 설정 다시 로드
alias reload='source ~/.bashrc'

이것들을 추가한 후 source ~/.bashrc를 실행하면 즉시 활성화된다.

과소평가된 하나:

alias -- -='cd -'

이전 폴더로 돌아간다. 브라우저 뒤로 버튼 같다. 나는 이걸 매일 쓴다.

트릭 #5: 아마 무시하고 있는 키보드 단축키

터미널은 아무도 알려주지 않아서 대부분의 개발자가 배우지 못한 키보드 단축키로 가득하다. 내가 자주 쓰는 것들:

Ctrl + A     →   줄 처음으로 이동
Ctrl + E     →   줄 끝으로 이동
Ctrl + W     →   한 단어 뒤로 삭제
Ctrl + U     →   커서 앞 모두 지우기
Ctrl + K     →   커서 뒤 모두 지우기
Ctrl + L     →   화면 지우기 ('clear'와 동일)
Ctrl + C     →   현재 명령 취소
Ctrl + Z     →   현재 프로세스 일시정지 ('fg'로 재개)
Alt + F      →   한 단어 앞으로 이동
Alt + B      →   한 단어 뒤로 이동

내가 가장 많이 쓰는 조합: Ctrl+A로 줄 처음으로 이동 후 Ctrl+K로 전부 삭제. 백스페이스를 누르고 있는 것보다 빠르다.

긴 명령을 수정할 때 단어 단위로 이동하는 Alt+F와 Alt+B는 게임 체인저다.

트릭 #6: tmux — 터미널 창이 꺼져도 작업이 죽지 않게

원격 서버에서 작업하거나 창 전환 없이 여러 터미널 분할 창을 원한다면, tmux가 모든 것을 바꾸는 도구다.

중요한 이유: tmux 없이는 SSH 연결이 끊기면 프로세스가 죽는다. tmux에서는 계속 실행된다. 재연결하면 정확히 떠난 곳에서 계속할 수 있다.

# 이름 있는 세션 시작
tmux new -s myproject

# 세션에서 분리 (모든 것이 계속 실행됨)
Ctrl + B, then D

# 나중에 재연결
tmux attach -t myproject

# 모든 세션 나열
tmux ls

# tmux 내부 - 창 분할
Ctrl + B, then %      # 수직 분할
Ctrl + B, then "      # 수평 분할
Ctrl + B, then arrow  # 창 간 이동

한 창에서 서버가 실행되고, 다른 창에서 로그가 스트리밍되고, 세 번째 창에 에디터가 있다. 같은 윈도우 안에서. SSH 탭 다섯 개를 띄워놓던 개발자들이 tmux를 보면 다시는 그렇게 안 한다.

트릭 #7: 백그라운드에서 실행하기

빌드, 테스트 스위트, 서버 같은 긴 프로세스를 실행한다. 새 탭을 열고 싶지 않다. 명령 끝에 &만 붙이면 된다:

npm run build &

프로세스가 백그라운드에서 실행된다. 터미널이 즉시 자유로워진다. 작업 ID가 출력된다.

fg          # 포그라운드로 가져오기
jobs        # 모든 백그라운드 작업 보기

# 이미 & 없이 시작한 프로세스는?
Ctrl + Z    # 일시정지
bg          # 백그라운드에서 재개

트릭 #8: 'cd + ls'를 하나의 명령으로 만들기

어딘가로 cd를 하고 바로 ls를 타이핑하는 패턴을 얼마나 자주 하는지 알게 될 것이다. 두 명령 대신 하나로 만들어라. .bashrc 또는 .zshrc에 추가:

function cl() {
  cd "$1" && ls -la
}

이제 cl my-project가 폴더로 이동하고 한 번에 내용을 나열한다. 쉘 함수는 인수를 받고 로직을 실행할 수 있어 별칭보다 강력하다.

트릭 #9: fzf — 모든 것을 퍼지 검색 가능하게 만드는 도구

이 목록에서 지금 당장 설치할 도구 하나를 골라야 한다면, fzf다.

fzf는 커맨드 라인용 퍼지 파인더다. 파일, 명령 히스토리, git 브랜치, 실행 중인 프로세스 — 목록인 것은 무엇이든 검색할 수 있다.

# macOS
brew install fzf

# Ubuntu / Debian
sudo apt install fzf

설치하면 Ctrl+R이 강력한 인터랙티브 히스토리 검색이 된다. 무엇이든 파이프로 넣을 수 있다:

# 파일을 퍼지 검색하고 열기
vim $(fzf)

# git 브랜치를 인터랙티브하게 전환
git checkout $(git branch | fzf)

# 프로세스를 인터랙티브하게 종료
kill $(ps aux | fzf | awk '{print $2}')

브랜치 전환이 내가 매일 쓰는 것이다. 더 이상 정확한 브랜치 이름을 기억하려고 애쓰지 않아도 된다.

트릭 #10: pbcopy / xclip으로 파이프 — 텍스트를 손으로 선택하지 마라

명령 출력을 클립보드에 복사하고 싶다. 보통 방법: 명령을 실행하고, 수동으로 출력을 선택하고, 우클릭, 복사. 터미널 방식:

# macOS
cat some-file.txt | pbcopy

# Linux
cat some-file.txt | xclip -selection clipboard

# SSH 공개 키 복사
cat ~/.ssh/id_rsa.pub | pbcopy

이제 출력이 즉시 클립보드에 있다. 마우스 필요 없다. SSH 키, API 토큰, 빌드 출력, 또는 다른 곳에 붙여넣어야 하는 긴 문자열에 매우 유용하다.

트릭 #11: 명령 자동 수정

잘못된 명령을 입력할 때가 있다. thefuck을 사용해 보라:

sudo apt install thefuck

그런 다음 실행:

fuck

마지막 명령을 제안하고 수정한다. 네, 실제 이름이다. 그리고 네, 유용하다.

진정한 변화

내가 깨달은 것이 있다: 대부분의 개발자는 터미널을 자판기처럼 대한다. 명령어를 치고, 결과를 받고, 반복한다.

하지만 터미널에서 능숙해 보이는 개발자들은 터미널과 관계를 쌓았다. 커스터마이징하고, 단축키를 배우고, 시간이 지남에 따라 복리를 낳는 작은 도구들에 투자했다.

이 트릭들 중 어느 것도 어렵지 않다. 설정하는 데 약 10분이 걸린다. 보상은 수개월, 수년에 걸친 더 부드러운 작업이다.

하나부터 시작하라. 아직 쓰지 않는다면 Ctrl+R. 설정하지 않았다면 별칭. 가장 큰 단일 업그레이드를 원한다면 fzf.

터미널은 도구다. 하지만 도구는 그것을 아는 손에 반응한다.
