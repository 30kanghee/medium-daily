Spark Structured Streaming: Output Modes Explained with Real-World Examples

Kaizen · 3 min read · 원문 · freedium

Spark Structured Streaming의 세 가지 출력 모드(Append, Update, Complete)를 실제 예제와 함께 명확하게 이해한다.

Spark Structured Streaming에서 출력 모드는 각 마이크로배치 이후 어떤 데이터를 싱크에 쓸지 결정하는 핵심 설정이다. Append 모드는 새로 추가된 행만 출력하며 집계 없는 스트림에 적합하고, Update 모드는 변경된 행만 기록해 대규모 상태 기반 처리에 효율적이다. Complete 모드는 매 배치마다 전체 결과 테이블을 다시 쓰므로 디버깅이나 소규모 데이터에 적합하다. 잘못된 출력 모드 선택은 런타임 오류나 심각한 성능 저하로 이어질 수 있어 운영 환경에서 특히 중요하다. 실시간 주문 대시보드 같은 실제 시나리오에서는 Update 모드가 가장 실용적인 선택이다.

지금까지 이 시리즈에서는 기초, 스트리밍 조인, 상태 기반 집계를 다뤘다. 이번에는 겉으로는 단순해 보이지만 인터뷰와 운영 시스템 모두에서 매우 중요한 개념으로 넘어간다.

Structured Streaming의 출력 모드

많은 스트리밍 버그와 성능 문제는 잘못된 출력 모드 선택에서 비롯된다. 이 글은 각 모드가 무엇인지뿐만 아니라 언제, 왜 사용해야 하는지를 이해하는 데 도움이 될 것이다.

1. 출력 모드란 무엇인가?

스트리밍에서 데이터는 마이크로배치 단위로 지속적으로 처리된다. 각 배치 이후 Spark는 다음을 결정해야 한다.

결과의 어느 부분을 출력에 써야 하는가?

이 결정을 제어하는 것이 바로 출력 모드이다.

2. 세 가지 출력 모드

Spark Structured Streaming은 세 가지 출력 모드를 지원한다.

  • Append (추가)
  • Update (업데이트)
  • Complete (완전)

각각의 동작 방식은 매우 다르다.

3. Append 모드

정의:

마지막 마이크로배치 이후 추가된 새로운 행만 기록된다.

예시:

df.writeStream \
  .format("console") \
  .outputMode("append")

동작:

새로운 행만 기록된다.

중요 규칙:

Append 모드는 행이 나중에 변경되지 않는 경우에만 작동한다.

작동하는 경우:

  • 집계 없는 이벤트 스트림
  • 워터마크가 있는 윈도우 집계

실패하는 경우:

  • 전역 집계 (예: groupBy("category"))
  • 카운트가 계속 업데이트되기 때문

4. Update 모드

정의:

마지막 배치 이후 변경된 행만 기록된다.

예시:

df.writeStream \
  .format("console") \
  .outputMode("update")

동작:

변경되거나 새로운 행만 기록된다.

핵심 장점:

  • Complete 모드보다 효율적
  • 전체 데이터셋을 다시 쓰는 것을 방지

사용 사례:

  • 집계
  • 상태 기반 처리
  • 업데이트가 빈번한 경우

5. Complete 모드

정의:

매 마이크로배치 이후 전체 결과 테이블이 기록된다.

예시:

df.writeStream \
  .format("console") \
  .outputMode("complete")

동작:

출력:

  • 매번 전체 테이블이 출력된다

주요 특징:

  • 항상 최신 전체 상태를 표시
  • 대규모 데이터셋에 비용이 많이 듦

사용 사례:

  • 디버깅
  • 소규모 데이터셋
  • 학습 목적

6. 올바른 출력 모드 선택하기

대부분의 사람들이 어려움을 겪는 부분이다.

의사결정 표:

7. 출력 모드 vs 상태 기반 연산

무상태 연산(Stateless Operations):

  • Append 모드가 완벽하게 작동

상태 기반 연산(Stateful Operations):

  • Update 또는 Complete 모드 필요

이유는?

  • 결과가 계속 변하기 때문

8. 실제 사례

시나리오: 실시간 주문 대시보드

원하는 것:

  • 카테고리별 주문 수 집계
  • 몇 초마다 업데이트

옵션 1: Complete 모드

  • 구현이 쉬움
  • 확장성이 없음

옵션 2: Update 모드 (더 나은 선택)

  • 변경된 카테고리만 업데이트
  • 대시보드에 효율적

옵션 3: Append 모드

  • 다음 조건에서만 작동:
  • window + watermark 사용 시
  • 이후에 결과가 변경되지 않는 경우

9. 자주 하는 실수

1. 집계에 Append 모드 사용

결과:

  • 런타임 오류 발생

2. 운영 환경에서 Complete 모드 사용

결과:

  • 높은 컴퓨팅 비용
  • 성능 문제

3. 워터마크 요구사항 무시

Append 모드에는 다음이 필요하다:

  • 확정된 결과
  • 그렇지 않으면 Spark가 무엇이 "새것"인지 결정할 수 없음

10. 인터뷰 관점

이 주제는 인터뷰에서 거의 반드시 나오는 질문이다.

자주 나오는 질문들:

  • Append, Update, Complete 모드의 차이
  • Append 모드가 집계와 함께 작동하지 않는 이유
  • 대규모 파이프라인에 가장 좋은 모드
  • 워터마크와 Append 모드의 관계

모범 답변 팁:

Append 모드는 새로운 행만 출력하고, Update 모드는 변경된 행을 출력하며, Complete 모드는 전체 결과를 출력합니다. 선택은 데이터가 불변인지, 집계가 사용되는지, 그리고 시간이 지남에 따라 결과가 변경될 수 있는지에 따라 달라집니다.

11. 핵심 정리

  • 출력 모드는 각 마이크로배치 이후 무엇이 기록되는지를 제어한다
  • Append 모드는 효율적이지만 제한이 있다
  • Update 모드는 실제 시스템에서 가장 실용적이다
  • Complete 모드는 주로 디버깅용이다
  • 잘못된 모드 선택은 파이프라인을 망가뜨릴 수 있다

12. 다음에 배울 것

출력 모드를 이해했다면, 다음으로 중요한 주제들은:

  • 트리거 유형 (처리 시간 vs 한 번 vs 연속)
  • Exactly-once 보장에 대한 심화 학습
  • 엔드투엔드 스트리밍 아키텍처 설계

출력 모드는 작은 설정 세부 사항처럼 보일 수 있지만, 실제로는 전체 스트리밍 파이프라인의 동작 방식을 정의한다. 이를 마스터하는 것은 신뢰할 수 있고 확장 가능한 실시간 시스템을 구축하는 데 필수적이다.
