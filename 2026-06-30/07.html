This Is Why SAP Acquired Dremio — and What It Means for Data Architects

Vinayak Gole · Towards Data Engineering · 6분 읽기 · 원문 · freedium

SAP의 Dremio 인수는 독점적 데이터 사일로 시대의 종언을 알리는 오픈 레이크하우스 시대의 서막이다.

요약

SAP가 오픈 데이터 레이크하우스의 선구자인 Dremio를 인수한다고 공식 발표했다. Dremio는 Apache Iceberg, Arrow, Polaris 같은 오픈소스 표준 위에 구축되어, 데이터를 이동시키지 않고도 클라우드 스토리지에 직접 고성능 SQL 쿼리를 실행할 수 있게 해준다. 이번 인수로 SAP Business Data Cloud(BDC)는 Iceberg 네이티브 플랫폼으로 진화하며, SAP 데이터와 외부 데이터 레이크가 하나의 오픈 기반 위에 공존하게 된다. 앞서 인수한 마스터 데이터 관리(MDM) 솔루션 Reltio와의 결합으로, SAP는 데이터 품질과 데이터 단편화라는 엔터프라이즈 AI의 두 가지 핵심 난제를 동시에 해결하는 독보적인 플랫폼을 갖추게 됐다. 결과적으로 이번 인수는 독점적 사일로 시대의 마감을 선언하고, 연합·오픈·에이전틱 인텔리전스로의 전환을 가속화하는 분수령이 될 것이다.

SAP와 Dremio 인수의 배경

SAP가 오픈 데이터 레이크하우스의 선구자 Dremio를 인수하겠다고 공식 발표했다. 이는 단순한 기업 간 거래가 아니다. 세계 최대 규모의 조직들이 앞으로 데이터를 어떻게 관리하고, 쿼리하고, 수익화할지를 근본적으로 바꾸는 전략적 전환점이다. 수십 년 동안 SAP 고객들은 고성능이지만 엄격하게 구조화된 SAP HANA의 세계와, 광대하지만 정돈되지 않은 클라우드 데이터 레이크 사이에서 이분법적인 선택을 강요받아 왔다. 이제 그 벽이 허물어지고 있다. Dremio를 핵심 생태계에 통합함으로써, SAP는 독점적 데이터 사일로의 시대가 저물고 있음을 공식적으로 선언한 셈이다.

에이전틱 레이크하우스: Dremio란 무엇인가

이번 인수의 파급력을 제대로 이해하려면 먼저 Dremio가 어떤 기술인지 알아야 한다. '데이터 레이크를 기존 데이터베이스만큼 쉽게 사용할 수 있게 한다'는 철학으로 설립된 Dremio는 '에이전틱 레이크하우스'라고도 불리며, 처음부터 끝까지 오픈소스 표준 위에 구축되어 있다. 테이블 포맷으로는 Apache Iceberg를, 인메모리 처리에는 Apache Arrow를, 데이터 카탈로그에는 Apache Polaris를 활용한다. 사실 Dremio는 Arrow와 Polaris 두 프로젝트 모두의 공동 창시자이기도 하다.

기존 데이터 웨어하우스는 데이터를 사용하기 전에 반드시 이동하고 변환해야 하는 반면, Dremio는 데이터가 있는 곳에서 직접 쿼리할 수 있다. Amazon S3든, Azure Data Lake Storage든, Google Cloud Storage든 관계없이 Dremio는 고성능 SQL 인터페이스를 통해 어디서나 데이터에 접근한다. 복잡한 ETL(추출·변환·적재) 파이프라인이 필요 없고, 데이터를 컴퓨팅 쪽으로 옮기는 대신 컴퓨팅을 데이터 쪽으로 가져오는 방식이다.

Dremio의 핵심 기능

Dremio의 가치는 엔터프라이즈 분석에서 가장 흔하게 발생하는 병목을 해결하는 다섯 가지 기술적 기둥에 있다.

1. 페더레이션 쿼리 (Federated Queries)

물리적으로 다른 위치에 존재하는 데이터셋을 조인할 수 있는 기능이다. 온프레미스 데이터베이스의 고객 테이블과 클라우드 S3 버킷의 클릭스트림 데이터를, 데이터 이동 없이 단 하나의 SQL 명령으로 결합할 수 있다.

2. Dremio Reflections

특허받은 쿼리 가속 기술이다. 쿼리 속도를 높이기 위해 데이터의 물리적 표현을 자동으로 생성하고 유지관리한다. 구조화된 뷰(Materialized View)와 유사하지만 더욱 지능적으로 작동하며, 사용자가 인프라를 직접 관리하지 않아도 수십억 행 규모의 데이터셋에서 1초 미만의 응답 시간을 실현한다.

3. 시맨틱 레이어 (Semantic Layer)

데이터에 대한 통합된 단일 뷰를 제공한다. 계산 필드나 보안 권한과 같은 비즈니스 로직을 한 번만 정의하면, Power BI, Tableau, AI 에이전트 등 어떤 도구를 사용하든 일관되게 적용된다. 이를 통해 조직 전체에 걸쳐 진정한 단일 정보 출처를 보장한다.

4. Iceberg 네이티브 지원

Apache Iceberg는 오픈 테이블 포맷의 산업 표준으로 자리 잡았다. Iceberg를 채택함으로써 Dremio는 여러 엔진이 동시에 데이터에 접근할 수 있도록 보장한다. 벤더에 종속될 위험이 없으며, 데이터는 Iceberg 프로토콜을 지원하는 모든 도구에서 읽을 수 있는 개방형 포맷으로 유지된다.

SAP Business Data Cloud(BDC)의 등장

Dremio가 어디에 맞물리는지 이해하려면 SAP의 데이터 전략이 진화해 온 방향, 즉 SAP Business Data Cloud(BDC)를 살펴봐야 한다. SAP BDC는 비즈니스 데이터 패브릭(Business Data Fabric)의 다음 세대다. SAP와 비SAP 데이터를 하나로 조화시키기 위해 설계된 포괄적인 클라우드 네이티브 플랫폼이다.

SAP BDC는 레거시 시스템을 대체하는 것이 아니라, SAP 핵심 기술들의 슈퍼셋이다. 데이터 모델링을 위한 SAP Datasphere, 시각화를 위한 SAP Analytics Cloud, 그리고 최근 추가된 머신러닝 플랫폼 SAP Databricks까지 통합한다. BDC의 목표는 SAP 애플리케이션의 풍부한 비즈니스 컨텍스트를 보존하면서도 현대 AI의 요구에 맞게 확장 가능한 통합 시맨틱 레이어를 제공하는 것이다.

엔터프라이즈 AI가 멈추는 이유는 모델이 부족해서가 아니라, 데이터가 AI 에이전트를 위해 준비되어 있지 않기 때문입니다. — SAP SE CTO, Philipp Herzig

완벽한 결합: Dremio가 SAP 비전에 맞는 이유

Dremio 인수는 SAP BDC 퍼즐의 또 하나의 핵심 조각이다. 그동안 SAP 고객들은 ERP 데이터를 외부 데이터 레이크와 통합하는 데 오랫동안 어려움을 겪어왔다. 그 과정은 느리고 비용이 많이 들었으며, SAP 데이터를 가치 있게 만드는 메타데이터가 손실되는 경우도 많았다.

Dremio는 SAP BDC를 Iceberg 네이티브 플랫폼으로 만들어 이 문제를 해결한다. SAP 데이터와 비SAP 데이터가 동일한 오픈 기반 위에서 공존할 수 있게 되며, 포맷 변환이 더 이상 필요하지 않다. 물리적인 데이터 통합 없이도 전체 데이터 랜드스케이프에 걸쳐 고성능 쿼리를 실행할 수 있다.

Dremio를 내장함으로써 SAP는 고객에게 Apache Polaris 기반의 범용 카탈로그를 제공하게 된다. 이 카탈로그는 엔터프라이즈 전체의 데이터 검색 계층으로 기능하며, SAP 시스템과 외부 레이크 전반에 걸쳐 데이터 계보, 접근 권한, 상호 관계를 추적한다. 이 카탈로그는 궁극적으로 SAP Knowledge Graph의 기반이 되어, AI 에이전트가 데이터 자체뿐만 아니라 그것을 둘러싼 조직 계층 구조와 규정 분류 체계까지 이해할 수 있도록 할 것이다.

경제적 파급 효과도 크다. Dremio의 서버리스 아키텍처는 탄력적 확장을 가능하게 한다. 수요가 많을 때는 자동으로 스케일 업되고, 유휴 상태에서는 스케일 다운되어 고정 용량의 데이터 웨어하우스를 유지하는 것보다 총소유비용(TCO)을 크게 줄여준다.

시너지 효과: Dremio와 Reltio의 협업

Dremio 인수는 마스터 데이터 관리(MDM) 분야의 선두주자 Reltio 인수에 뒤이어 이루어진 것이다. SAP 분석의 미래를 이해하려면 이 두 기술이 어떻게 맞물리는지 살펴봐야 한다.

Reltio는 SAP Business Data Cloud의 토대를 담당한다. 황금 레코드(golden record)를 생성하고, 여러 시스템에 걸쳐 고객·공급업체·제품과 관련된 데이터를 정제하고 통합하는 역할이다. Reltio 덕분에 고객 레코드를 조회할 때 항상 단 하나의 정확한 버전의 진실을 볼 수 있다.

Dremio는 그 토대 아래 계층, 즉 데이터 접근 및 레이크하우스 계층에 자리한다. Reltio가 데이터의 품질을 보장한다면, Dremio는 그 데이터를 대규모로 접근하고 쿼리할 수 있도록 보장한다. SAP 고객 입장에서는 Reltio로 핵심 비즈니스 엔터티를 정의하고, Dremio로 그 황금 레코드를 레이크에 저장된 방대한 원시 텔레메트리나 소셜 미디어 데이터와 연결할 수 있게 되는 것이다.

이 조합은 경쟁자들에게 치명적이다. 엔터프라이즈 AI에서 가장 어려운 두 가지 문제, 즉 데이터 품질과 데이터 단편화를 동시에 해결하기 때문이다. Reltio와 Dremio를 갖춘 SAP는 파편화된 원시 데이터부터 거버넌스가 완비된 AI 준비 인텔리전스까지, 단일 오픈 플랫폼에서 완결되는 엔드투엔드 스택을 확보하게 됐다.

SAP 고객을 위한 분석의 미래

이것이 SAP 고객에게 의미하는 바는 무엇인가? 로드맵이 바뀌었다. SAP 생태계에서 분석의 미래는 더 이상 더 큰 사일로를 쌓는 것이 아니다. 연합적이고, 개방적이며, 에이전틱한 인텔리전스가 그 지향점이다.

가장 즉각적인 혜택은 마이그레이션 부담의 해소다. 규제나 비용 문제로 특정 데이터셋을 SAP 환경으로 이전하기를 꺼려온 조직들이 많다. Dremio를 활용하면 데이터를 그 자리에 그대로 두면서도 SAP 비즈니스 컨텍스트 안으로 끌어들일 수 있다.

Dremio는 마이그레이션의 번거로움 없이 SAP 고객이 비SAP 데이터에 접근할 수 있게 합니다. — BARC US 리서치 부사장, Kevin Petrie

장기적으로 이번 인수는 SAP 생태계 내 에이전틱 AI의 길을 열어준다. AI 에이전트가 실시간으로 고품질 데이터에 접근해야 의사결정이 가능하다. 주 1회 ETL 작업이 끝나기를 기다릴 여유가 없다. Dremio의 1초 미만 SQL 엔진과 Iceberg 네이티브 아키텍처가 바로 그 저지연 접근 환경을 제공한다. 공급망 재고를 최적화하는 에이전트든, 금융 사기를 탐지하는 에이전트든, 모두 Dremio의 연합적 쿼리 능력에 의존하게 될 것이다.

나아가 SAP 고객들은 더 많은 선택의 자유를 누리게 된다. 플랫폼이 오픈 표준 위에 구축되어 있으므로 원하는 분석 도구를 무엇이든 사용할 수 있다. SAP 전용 도구만 써야 한다는 제약이 사라지는 것이다. 이는 SAP 스택에 대한 가장 큰 역사적 불만 중 하나였던 독점 데이터 종속 문제를 해소한다.

Dremio 인수는 SAP가 AI 패권 경쟁에서 독점적 경계보다 개방성과 성능을 우선시한다는 명확한 신호입니다. — 이번 거래를 담당한 업계 애널리스트

마무리: 분수령이 된 인수

SAP의 Dremio 인수는 분수령이 될 사건이다. 데이터 웨어하우스의 시대에서 오픈 데이터 레이크하우스의 시대로 전환되는 역사적 전환점이다. SAP 고객 입장에서는 비용 절감, 빠른 가치 실현, 자율적이고 AI 주도적인 비즈니스 프로세스로 나아가는 명확한 경로가 열리는, 압도적으로 긍정적인 발전이다.

Dremio를 SAP Business Data Cloud에 통합함으로써 SAP는 데이터 단편화 문제를 단번에 해결하는 플랫폼을 제공하게 됐다. SAP 데이터와 비SAP 데이터가 대등하게 공존하는 환경, Dremio의 제자리 쿼리 능력과 Reltio의 마스터 데이터 거버넌스의 결합은 시장에서 유례없는 경쟁력이다.

2026년에 출시될 Business Data Cloud는 이전의 어떤 것과도 근본적으로 다르다. 더 빠르고, 더 개방적이며, 에이전트 시대를 위한 준비가 되어 있다. SAP 데이터를 둘러싼 벽이 단순히 무너진 것이 아니라, 엔터프라이즈 전체로 나아가는 관문으로 재건된 것이다.
