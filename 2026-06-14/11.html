5 LangChain & LangGraph Concepts You Should Know

anthony-kigotho · Level Up Coding · 7 min read · 원문 · freedium

LangChain과 LangGraph를 처음 접하는 개발자라면 꼭 알아야 할 5가지 핵심 개념을 — 채팅 모델 호출 방식부터 에이전트, 체인, 상태 관리, 스트리밍까지 — 실용적인 코드 예제와 함께 설명한다.

요약

LangChain은 LLM 애플리케이션을 구축하기 위한 가장 인기 있는 오픈소스 프레임워크 중 하나이며, LangGraph는 그 위에 구축된 그래프 기반 워크플로 엔진으로 상태 저장, 멀티 에이전트 시스템, 조건부 라우팅을 가능하게 한다. 이 글은 채팅 모델을 호출하는 다양한 방식(invoke, stream, batch), 체인을 구성하는 LCEL, LangGraph의 노드와 엣지, 도구 호출 에이전트 패턴, 메모리 관리까지 5가지 개념을 실용적 코드 예제와 함께 설명한다. LangChain과 LangGraph를 함께 이해하면 간단한 RAG 파이프라인부터 복잡한 멀티 에이전트 시스템까지 폭넓은 AI 애플리케이션을 구축할 수 있는 탄탄한 기반이 마련된다.

1. 채팅 모델을 호출하는 다양한 방식

LangChain에서 채팅 모델을 사용할 때는 세 가지 주요 호출 방식이 있다. invoke는 동기식으로 단일 응답을 반환하는 기본 방식이다. stream은 토큰이 생성되는 대로 스트리밍해 실시간 응답 경험을 제공한다. batch는 여러 입력을 한 번에 처리해 처리량을 높인다.

from langchain_openai import ChatOpenAI

llm = ChatOpenAI(model="gpt-4o")

# 기본 호출
response = llm.invoke("LangChain이 무엇인가요?")
print(response.content)

# 스트리밍
for chunk in llm.stream("LangGraph를 설명해 주세요."):
    print(chunk.content, end="", flush=True)

# 배치 처리
responses = llm.batch(["질문 1", "질문 2", "질문 3"])

챗봇처럼 실시간 피드백이 중요한 경우에는 stream을, 대량의 데이터를 처리할 때는 batch를, 그 외 일반적인 경우에는 invoke를 사용하면 된다.

2. LCEL과 체인 구성

LCEL(LangChain Expression Language)은 파이프(|) 연산자를 사용해 컴포넌트를 체인으로 연결하는 선언적 방식이다. 프롬프트 템플릿, 모델, 출력 파서를 간결하게 조합할 수 있으며, 자동으로 스트리밍과 배치를 지원한다.

from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser
from langchain_openai import ChatOpenAI

prompt = ChatPromptTemplate.from_template(
    "{topic}에 대해 한 문장으로 설명해 주세요."
)
model = ChatOpenAI(model="gpt-4o")
parser = StrOutputParser()

# 파이프로 체인 구성
chain = prompt | model | parser

result = chain.invoke({"topic": "LangGraph"})
print(result)

LCEL의 핵심 장점은 구성 요소들이 모두 동일한 Runnable 인터페이스를 따르기 때문에 자유롭게 조합하고 재사용할 수 있다는 점이다.

3. LangGraph의 노드와 엣지

LangGraph는 워크플로를 방향성 그래프로 모델링한다. 노드(Node)는 실제 작업을 수행하는 Python 함수이고, 엣지(Edge)는 노드 간의 전환 규칙을 정의한다. 조건부 엣지를 사용하면 LLM의 출력에 따라 다른 경로로 분기할 수 있다.

from langgraph.graph import StateGraph, END
from typing import TypedDict

class State(TypedDict):
    question: str
    answer: str

def answer_node(state: State) -> State:
    answer = llm.invoke(state["question"]).content
    return {"answer": answer}

def should_continue(state: State) -> str:
    if len(state["answer"]) < 50:
        return "retry"
    return END

graph = StateGraph(State)
graph.add_node("answer", answer_node)
graph.add_conditional_edges("answer", should_continue)
graph.set_entry_point("answer")
app = graph.compile()

이 구조를 통해 루프, 분기, 병렬 실행 등 복잡한 워크플로를 직관적으로 표현할 수 있다.

4. 도구 호출 에이전트

에이전트는 LLM이 어떤 도구를 어떤 순서로 호출할지 스스로 결정하는 시스템이다. LangChain의 @tool 데코레이터로 함수를 도구로 만들고, LangGraph의 create_react_agent를 사용하면 ReAct 패턴의 에이전트를 쉽게 구축할 수 있다.

from langchain_core.tools import tool
from langgraph.prebuilt import create_react_agent

@tool
def search_web(query: str) -> str:
    """웹에서 정보를 검색합니다."""
    return f"{query}에 대한 검색 결과..."

@tool
def calculator(expression: str) -> str:
    """수학 표현식을 계산합니다."""
    return str(eval(expression))

tools = [search_web, calculator]
agent = create_react_agent(llm, tools)

result = agent.invoke({
    "messages": [("user", "파리의 현재 인구를 검색하고 서울 인구의 몇 퍼센트인지 계산해 주세요.")]
})

에이전트는 도구들을 적절한 순서로 호출해 복잡한 작업을 자동으로 처리한다. 중요한 것은 도구의 독스트링이 LLM이 해당 도구를 언제 사용할지 판단하는 데 핵심 역할을 한다는 점이다. 명확하고 구체적인 설명을 작성하는 것이 좋다.

5. 상태 관리와 메모리

LangGraph의 가장 강력한 기능 중 하나는 내장된 상태 관리다. MemorySaver를 사용하면 여러 대화 세션에 걸쳐 상태를 유지할 수 있어, 이전 대화를 기억하는 챗봇을 쉽게 만들 수 있다.

from langgraph.checkpoint.memory import MemorySaver
from langgraph.prebuilt import create_react_agent

memory = MemorySaver()
agent = create_react_agent(llm, tools, checkpointer=memory)

# thread_id로 대화 세션 구분
config = {"configurable": {"thread_id": "user-123"}}

# 첫 번째 메시지
agent.invoke(
    {"messages": [("user", "내 이름은 김철수입니다.")]},
    config=config
)

# 두 번째 메시지 - 이전 대화를 기억
result = agent.invoke(
    {"messages": [("user", "내 이름이 뭐라고 했죠?")]},
    config=config
)
# "김철수"라고 올바르게 응답

thread_id를 통해 서로 다른 사용자의 대화를 격리하고 각 세션의 컨텍스트를 독립적으로 유지할 수 있다. 프로덕션 환경에서는 MemorySaver 대신 PostgreSQL이나 Redis 기반의 영속성 체크포인터를 사용하는 것이 권장된다.

마치며

이 다섯 가지 개념 — 채팅 모델 호출 방식, LCEL 체인, LangGraph 노드/엣지, 도구 호출 에이전트, 상태 관리 — 은 LangChain과 LangGraph 생태계를 이해하는 핵심 기반이다. 이를 바탕으로 간단한 RAG 시스템부터 복잡한 멀티 에이전트 시스템까지 다양한 AI 애플리케이션을 구축할 수 있다.
