I Used the Terminal Wrong for Years

Tushar Kanjariya · 7 min read · 원문 · freedium

터미널을 단순 명령 입력창이 아닌 환경으로 다루는 순간 생산성이 달라진다 — 매일 쓰는 11가지 실전 트릭.

요약

저자는 시니어 개발자가 구글링도 없이 90초 만에 배포 문제를 해결하는 것을 보고 터미널을 제대로 쓰지 못했다는 사실을 깨달았다. 터미널은 명령을 입력하는 도구가 아니라 하나의 환경이다. !!, !$로 이전 명령 재활용, Ctrl+R로 히스토리 검색, ^old^new로 오타 즉시 수정, 별칭(alias) 설정, 키보드 단축키, tmux로 세션 유지, 백그라운드 실행, fzf 퍼지 검색, pbcopy/xclip으로 클립보드 파이프 등 11가지 기법을 소개한다. 이 기법들은 각각 몇 분이면 설정할 수 있고, 쌓이면 몇 달치 마찰을 없애준다.

시니어 개발자가 90초 안에 배포 문제를 해결하는 것을 봤습니다.

구글링도 없었습니다. 치트시트에서 복사 붙여넣기도 없었습니다. 터미널 안에서 마치 대화하듯 자연스럽게 흘러갔습니다.

저는 그가 특별한 도구를 쓴다고 생각했습니다. 아니었습니다. 그는 그냥 저보다 터미널을 잘 알았을 뿐이었습니다.

대부분의 개발자가 겪는 문제

대부분의 개발자는 처음 튜토리얼에서 배운 방식 그대로 터미널을 씁니다: 명령을 입력하고, 엔터를 치고, 반복합니다.

그건 작동합니다.

하지만 느립니다.

그리고 시간이 지날수록 그 작은 지연이 진짜 마찰이 됩니다.

제가 몰랐던 것은 이겁니다:

터미널은 단순한 도구가 아닙니다. 그것은 하나의 환경입니다.

그렇게 대하기 시작하면 모든 것이 달라집니다.

여기 제가 매일 실제로 쓰는 트릭들을 소개합니다. 50개 명령이 아닙니다. 기억에 남는 것들만입니다.

트릭 #1: 이미 실행한 명령을 다시 타이핑하지 마라

이걸 알았을 때 충격을 받았습니다.

긴 명령을 실행했는데 권한 오류로 실패했습니다. sudo를 붙여 다시 시도하려 합니다. 위쪽 화살표를 눌러 커서를 줄 앞으로 이동하는 대신 그냥 이렇게 치세요:

sudo !!

!!는 마지막 명령 전체로 확장됩니다. 이게 전부입니다. 전부 다시 타이핑하는 대신 두 글자.

이것도 좋습니다:

mkdir my-project
cd !$

!$는 이전 명령의 마지막 인자를 줍니다. 연산을 체이닝할 때 엄청나게 유용합니다. 폴더를 만들고 폴더 이름을 두 번 타이핑하지 않고 바로 진입합니다.

저는 !$를 하루에 약 20번 씁니다. 사소하지만 없애주는 마찰이 진짜입니다.

트릭 #2: Ctrl+R — 터미널의 검색 엔진

위쪽 화살표가 히스토리를 순환한다는 건 모두 압니다. 하지만 그건 선형입니다 — 원하는 걸 찾을 때까지 계속 위쪽을 눌러야 합니다.

Ctrl+R은 전체 히스토리에 대해 역방향 검색을 합니다. 명령의 일부를 입력하기 시작하면 가장 최근 일치 항목을 찾아줍니다.

(reverse-i-search)`docker': docker-compose up -d --build

Ctrl+R을 다시 눌러 더 오래된 일치 항목을 순환합니다. 엔터를 눌러 실행하거나, Esc를 눌러 먼저 편집합니다.

이 하나의 변화만으로도 매일 몇 분을 절약합니다. 긴 docker, git, ssh 명령을 반복해서 사용한다면, 없이 어떻게 살았는지 의문스러워질 겁니다.

보너스: 히스토리 크기를 늘려 더 멀리 검색할 수 있게 하세요. .bashrc 또는 .zshrc에 추가하세요:

HISTSIZE=10000
HISTFILESIZE=20000
HISTCONTROL=ignoredups:erasedups

히스토리가 10,000개 명령까지 늘어나고 중복으로 어수선해지지 않습니다. 검색이 실제로 유용해집니다.

트릭 #3: 오타 즉시 수정

오타가 있는 명령을 방금 실행했습니다. 전형적인 예:

git chekcout main

위쪽 화살표를 누르고 수동으로 편집하는 대신 ^old^new 치환 트릭을 사용하세요:

^chekcout^checkout

마지막 명령에서 chekcout을 checkout으로 교체하고 즉시 실행합니다.

팀원들이 멈추고 "잠깐, 어떻게 한 거야?"라고 물어볼 트릭입니다. 설명하면서 뿌듯함을 느낄 겁니다.

트릭 #4: 별칭(Alias)은 공짜 생산성 — 활용하라

일주일에 다섯 번 이상 타이핑하는 모든 명령은 별칭을 가질 자격이 있습니다.

별칭은 ~/.bashrc(Linux) 또는 ~/.zshrc(macOS)에 저장됩니다. 실제로 사용하는 것들의 스타터 팩입니다:

# 탐색
alias ..='cd ..'
alias ...='cd ../..'
alias ~='cd ~'
alias -- -='cd -'          # 이전 디렉토리로 돌아가기

# 목록 나열
alias ll='ls -alF'
alias lt='ls -ltr'         # 시간순으로 나열, 최신이 마지막

# 안전망 (덮어쓰기 전에 확인)
alias rm='rm -i'
alias cp='cp -i'
alias mv='mv -i'

# Git 단축키
alias gs='git status'
alias ga='git add .'
alias gc='git commit -m'
alias gp='git push'
alias gl='git log --oneline --graph --decorate'
alias gco='git checkout'

# Docker 단축키
alias dps='docker ps'
alias dc='docker-compose'
alias dcu='docker-compose up -d'
alias dcd='docker-compose down'

# 개발 단축키
alias ni='npm install'
alias nrd='npm run dev'
alias nrb='npm run build'

# 편집 후 쉘 설정 재로드
alias reload='source ~/.bashrc'

추가 후 source ~/.bashrc를 실행하면(또는 reload 별칭을 추가했다면 그걸로) 즉시 활성화됩니다.

과소평가된 것 하나:

alias -- -='cd -'

이전 폴더로 돌아갑니다. 브라우저 뒤로 가기 버튼 같은 것입니다. 저는 매일 씁니다.

트릭 #5: 아마 무시하고 있는 키보드 단축키

터미널에는 아무도 알려주지 않아서 대부분의 개발자가 배우지 못한 키보드 단축키들이 가득합니다. 제가 자주 쓰는 것들:

Ctrl + A     →   줄 맨 앞으로 이동
Ctrl + E     →   줄 맨 끝으로 이동
Ctrl + W     →   한 단어 뒤로 삭제
Ctrl + U     →   커서 앞의 모든 것 삭제
Ctrl + K     →   커서 뒤의 모든 것 삭제
Ctrl + L     →   화면 지우기 ('clear'와 동일)
Ctrl + C     →   현재 명령 취소
Ctrl + Z     →   현재 프로세스 일시 중단 ('fg'로 재개)
Alt  + F     →   한 단어 앞으로 이동
Alt  + B     →   한 단어 뒤로 이동

제가 가장 많이 쓰는 조합: Ctrl+A로 줄 맨 앞으로 점프한 다음 Ctrl+K로 전체를 삭제합니다. 백스페이스를 계속 누르는 것보다 빠릅니다.

Alt+F와 Alt+B로 단어 단위로 점프하는 건 긴 명령을 편집할 때 게임 체인저입니다. 더 이상 화살표 키를 누르고 있을 필요 없습니다.

트릭 #6: tmux — 터미널 창이 닫혀도 작업이 죽지 않게

원격 서버에서 작업하거나, 창을 전환하지 않고 여러 터미널 창을 원한다면 tmux가 모든 것을 바꿔주는 도구입니다.

왜 중요한가: tmux 없이는 SSH 연결이 끊어지면 프로세스가 죽습니다. tmux와 함께라면 계속 실행됩니다. 재연결하면 정확히 떠난 곳에서 이어집니다.

# 이름 있는 세션 시작
tmux new -s myproject

# 세션에서 분리 (모든 것이 계속 실행됨)
Ctrl + B, 그다음 D

# 나중에 재연결
tmux attach -t myproject

# 모든 세션 나열
tmux ls

tmux 안에서 터미널을 창으로 분할할 수 있습니다:

Ctrl + B, 그다음 %      # 세로로 분할
Ctrl + B, 그다음 "      # 가로로 분할
Ctrl + B, 그다음 화살표  # 창 간 전환

여기서 재미있어집니다. 한 창에서 서버를 실행하고, 다른 창에서 로그를 스트리밍하고, 세 번째 창에서 에디터를 사용할 수 있습니다 — 모두 같은 윈도우 안에서.

SSH 세션을 5개의 브라우저 탭으로 실행하는 개발자들을 봤습니다. tmux를 보고 나서는 다시 돌아가지 않았습니다.

트릭 #7: 백그라운드에서 실행하기

빌드, 테스트 슈트, 서버 같은 긴 프로세스를 실행 중입니다. 새 탭을 열고 싶지 않습니다. 명령 뒤에 &를 붙이면 됩니다:

npm run build &

프로세스가 백그라운드에서 실행됩니다. 터미널이 즉시 자유로워집니다. [1] 23456과 같은 작업 ID가 출력됩니다.

포그라운드로 가져오려면:

fg

모든 백그라운드 작업을 보려면:

jobs

& 없이 이미 긴 프로세스를 시작했다면, Ctrl+Z로 일시 중단한 다음 백그라운드에서 재개할 수 있습니다:

Ctrl + Z      # 일시 중단
bg            # 백그라운드에서 재개

트릭 #8: 'cd + ls'를 하나의 명령으로

cd somewhere 다음에 즉시 ls를 타이핑하는 패턴이 얼마나 자주인지 알게 될 겁니다.

두 명령 대신, 하나의 함수로 만드세요. .bashrc 또는 .zshrc에 추가하세요:

function cl() {
  cd "$1" && ls -la
}

이제 cl my-project는 폴더로 이동하고 한 번에 내용을 나열합니다.

셸 함수는 인자를 받을 수 있고 로직을 실행할 수 있기 때문에 별칭보다 더 강력합니다. 이것은 간단한 예이지만, 자신의 워크플로우에 맞춤화된 미니 도구를 구축하는 문을 열어줍니다.

트릭 #9: fzf — 모든 것을 퍼지 검색 가능하게

이 목록 전체에서 지금 당장 하나만 설치해야 한다면, fzf입니다.

fzf는 커맨드라인용 퍼지 파인더입니다. 파일, 명령 히스토리, git 브랜치, 실행 중인 프로세스 — 목록인 것이라면 무엇이든 검색할 수 있습니다.

# macOS
brew install fzf

# Ubuntu / Debian
sudo apt install fzf

설치하면 Ctrl+R이 단순한 역방향 검색 대신 강화된 인터랙티브 히스토리 검색이 됩니다. 모든 일치 항목을 한 번에 볼 수 있고, 실시간으로 필터링하고, 화살표로 탐색할 수 있습니다.

무엇이든 파이프로 넣을 수도 있습니다:

# 퍼지 검색으로 파일 열기
vim $(fzf)

# git 브랜치 인터랙티브하게 전환
git checkout $(git branch | fzf)

# 프로세스 인터랙티브하게 종료
kill $(ps aux | fzf | awk '{print $2}')

브랜치 전환이 제가 매일 쓰는 겁니다. 더 이상 정확한 브랜치 이름을 기억하려고 애쓸 필요 없습니다.

트릭 #10: pbcopy / xclip으로 파이프 — 텍스트 수동 선택 중지

명령 출력을 클립보드에 복사하고 싶습니다. 일반적인 방법: 명령 실행, 출력을 수동으로 선택, 오른쪽 클릭, 복사.

터미널 방식:

# macOS
cat some-file.txt | pbcopy

# Linux
cat some-file.txt | xclip -selection clipboard

이제 출력이 즉시 클립보드에 있습니다. 마우스가 필요 없습니다.

SSH 키, API 토큰, 빌드 출력, 또는 다른 곳에 붙여넣어야 하는 긴 문자열을 복사할 때 엄청나게 유용합니다.

# SSH 공개 키 복사
cat ~/.ssh/id_rsa.pub | pbcopy

명령 하나. 키가 클립보드에. 완료.

트릭 #11: 명령 자동 수정

때로는 잘못된 명령을 입력합니다. thefuck을 사용하세요:

sudo apt install thefuck

그다음 이렇게 실행합니다:

fuck

마지막 명령을 제안하고 수정해줍니다. 네, 실제 이름이 그렇습니다. 그리고 네, 유용합니다.

진짜 변화

제가 깨달은 것이 있습니다: 대부분의 개발자는 터미널을 자판기처럼 다룹니다. 명령을 입력하고, 출력을 받고, 반복합니다.

하지만 터미널에서 자연스러워 보이는 개발자들은 터미널과 관계를 쌓았습니다. 커스터마이징하고, 단축키를 배우고, 시간이 지날수록 복리로 불어나는 작은 도구들에 투자했습니다.

이 트릭들 중 어느 것도 어렵지 않습니다. 설정하는 데 약 10분이 걸립니다. 그 대가는 몇 달, 몇 년의 더 부드러운 작업입니다.

하나부터 시작하세요. 아직 쓰지 않는다면 Ctrl+R. 설정하지 않았다면 별칭. 단일 최대 업그레이드를 원한다면 fzf.

터미널은 도구입니다. 하지만 도구는 그것을 아는 손에 반응합니다.

읽어주셔서 감사합니다.
